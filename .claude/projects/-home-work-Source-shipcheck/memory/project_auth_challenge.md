인디해커/노코드 개발자 제품의 약 60-70%가 핵심 기능 사용에 로그인 필요. 빅테크(아마존, 오늘의집)와 달리 인디 SaaS는 "상호작용하는 도구"라 로그인 없이는 핵심 가치 체험 불가.

**Why:** 빅테크는 "구경할 수 있는 것"을 판매 (카탈로그 브라우징 가능), 인디 SaaS는 "사용해야 가치가 생기는 도구" (CRM, 분석, 폼빌더 등). 노코드 중에서도 Bubble/Softr/Glide(앱 빌더) 제품은 85-95%가 로그인 필요, 반면 Webflow/Framer/Carrd(사이트 빌더)는 대부분 공개.

**How to apply:**
- MVP에서는 Tier 1(공개 영역 "첫인상 시뮬레이션") + Tier 2(단일 테스트 계정 기반) 집중
- "URL만 입력하세요"의 순수함이 약간 손상되지만 치명적 블로커는 아님
- ICP에서 "이메일/비번 가입 + 무료 체험 있는 SaaS"까지 포함하면 전체 인디 제품의 약 50-60% 커버
- 검증 필요 가설: "랜딩→가입 전 경험 시뮬레이션만으로도 인디해커에게 충분한 가치를 주는가?"
