## 1. 문제인식 (Problem)

### 1-1. 창업아이템 배경 및 필요성

2026년 기준 새 앱의 75%가 노코드/로코드로 만들어진다. **제품을 만드는 것은 쉬워졌지만, "이걸 사람들이 진짜 원하는가?"를 확인하는 방법은 여전히 비싸고 느리다.**

| 현재 선택지 | 비용 | 한계 |
|---|---|---|
| 지인 3~5명 테스트 | 무료 | 편향. 다 "좋다"고 함 |
| 베타 테스터 모집 | $2,000~5,000 / 2주 | 솔로 파운더에게 비현실적 |
| UserTesting.com | ~$46,000/년 | 인디해커가 쓸 수 있는 가격 아님 |
| ChatGPT "리뷰해줘" | 무료 | **제품을 한 번도 써보지 않고 추측** |

> *"80명에게 개인적으로 공유했는데, 겨우 1명이 가입했다."* — Indie Hackers

연간 15,000개 이상의 웹 제품이 출시되고, 대부분은 충분한 사용자 피드백 없이 세상에 나간다. **핵심 Pain Point: "만드는 건 끝났는데, 검증할 돈도 시간도 없다."**

### 1-2. 창업아이템 목표시장(고객) 현황 분석

**목표 고객:** UX 리서처 없는 초기 스타트업, 인디해커, 솔로 빌더

| 시장 구분 | 규모 | 근거 |
|---|---|---|
| **TAM** (AI 합성 UX 검증 전체) | $150M~300M | 2024~26년 카테고리 VC 유입 $1.5B+ |
| **SAM** (인디/스타트업 웹앱) | $0.5M~1.0M (현재) | 연 15,000+ 웹제품 출시, 테스트 가능 ~10,000개 |
| **SOM** (1차 진입) | Product Hunt/IH 커뮤니티 | 마이크로 SaaS 시장 $15.7B → $59.6B (2030), CAGR 30% |

**경쟁 현황 — $1.5B+ 투자된 시장, 비어있는 포지션:**

| 회사 | 펀딩 | 접근법 | 실제 제품 사용? |
|---|---|---|:---:|
| **Simile** (Stanford) | $100M Series A | AI 디지털 트윈, 행동 예측 | No |
| **Aaru** | $50M+ ($1B 밸류에이션) | AI 설문/여론 시뮬레이션 | No |
| **Blok** | $7.5M Seed | Figma + Amplitude 데이터 기반 예측 | No |
| **Artificial Societies** (YC) | $5.35M | 대규모 네트워크 시뮬레이션 | No |
| **Personica** | **$0** | **실제 브라우저 조작 + 감정/인지 시뮬레이션** | **Yes** |

> 모든 경쟁사가 Figma, 설문, 기존 분석 데이터로 우회한다. **실제 제품을 AI가 사람처럼 사용하는 상용 서비스는 Personica뿐.**

핵심 차이:
- **Blok**: Amplitude/Mixpanel 데이터 필수 → 사용자가 없는 MVP에서는 작동 불가
- **Aaru/Synthetic Users**: 제품을 써보지 않고 인터뷰/설문 방식 → "써본 경험" 없음
- **Personica**: URL과 테스트 계정만 있으면 동작. 기존 데이터 불필요. **Pre-launch MVP에 최적**

## 2. 실현가능성 (Solution)

### 2-1. 창업아이템 현황 (준비 정도)

**핵심 엔진 개발 완료** (2026.01~03). 감정 파이프라인, 브라우저 자동화, 페르소나 분화가 동작하는 프로토타입 확보.

**작동 방식:** Personica에 URL과 타겟 유저 설명을 입력하면:
1. **AI 페르소나 생성** — 나이, 직업, 성격(Big Five), 기술 숙련도가 모두 다른 100~500명
2. **실제 브라우저 사용** — Playwright 기반으로 클릭, 입력, 탐색. 스크린샷 추측이 아닌 실사용
3. **감정/인지 시뮬레이션** — 클릭 실패 시 감정 엔진이 수치 조정, AI는 그 범위 안에서만 행동
4. **리포트 생성** — 출시 준비도(Go/조건부/No-Go), 이탈 지점 Top 3, 세그먼트별 만족도

**검증 완료 사항:**

| 항목 | 결과 |
|---|---|
| **페르소나 분화** | 같은 제품에서 성격에 따라 이탈 시점 5스텝 차이, 이탈 이유 완전 상이 (감정적 vs 분석적) |
| **감정 파이프라인 효과** | Ablation으로 입증 — 파이프라인 없으면 AI가 제품에 관대(sycophancy). PU 0.3→0.7로 왜곡 |
| **G2 리뷰 방향 일치** | Tally.so G2 리뷰 88개와 시뮬레이션 결과 주요 테마 방향적 일치 + 비영어권 장벽 추가 발견 |
| **원가 구조** | 실제 API 가격 기반 검증, 모든 시나리오에서 Gross Margin 78%+ |

**실제 시뮬레이션 로그 (Tally.so 폼 빌더):**

> **박서연 (52세 교사, 불안 성향)** — *"화면이 전부 영어라서 당황스럽다"* → 안전한 경로 탐색 → *"휴지통 아이콘 누르면 다 사라질 것 같아서 무섭다"* → **Step 7에서 자발적 이탈**
>
> **김도현 (31세 PM, 분석형)** — 에디터 직행 → 조건부 로직 확인 → *"조건부 로직이 드롭다운 3개가 전부잖아"* → **Step 12에서 기능 부족 판단 이탈**
>
> 같은 제품, 이탈 시점 5스텝 차이, 이유 완전히 다름. **이것이 세그먼트별 UX 인사이트.**

### 2-2. 창업아이템 실현 및 구체화 방안

**ChatGPT와의 핵심 차별화 — 수학적 감정 모델:**

| 지표 | 감정 파이프라인 있음 (Personica) | 없음 (ChatGPT 수준) |
|---|:---:|:---:|
| 제품 유용성 평가 (PU) | 0.3~0.5 | **0.6~0.8** (관대) |
| "포기하고 싶다" 언급 | 4회 | **0회** |
| 매몰 비용 심리 자연 발생 | 있음 | 없음 |

> 감정 파이프라인 없이는 AI가 항상 호의적. **솔직한 피드백이 아니라 칭찬을 받게 됨.**

**기술 구현:**
- **감정 엔진:** Big Five 성격 모델 → 수학적 매핑 → 클릭/실패 이벤트에 따른 감정 수치 실시간 조정
- **브라우저 자동화:** Playwright + Browserless 클라우드 → 실제 DOM 조작, 로그인/입력/탐색
- **LLM 통합:** GPT-4.1 mini 기반 행동 결정 + 감정 범위 제약 → sycophancy 방지
- **리포트 자동화:** SUS 사용성 점수, 세그먼트별 히트맵, 이탈 분석 자동 생성

## 3. 성장전략 (Scale-up)

### 3-1. 비즈니스 모델

**수익 모델:** 시뮬레이션 건당 과금 (Pay-per-run) + 구독 전환

| 플랜 | 가격 | 페르소나 수 | Gross Margin |
|---|---|---|---|
| **Starter** | $49/회 | 50명 | 89.6% |
| **Pro** | $99/회 | 200명 | 84.0% |
| **Scale** | $199/회 | 500명 | 81.3% |

**유닛 이코노믹스 (검증 완료):**

| 지표 | 수치 |
|---|---|
| Gross Margin | 78~94% (모든 시나리오) |
| CAC (커뮤니티 기반) | $15~25 |
| LTV/CAC | 12~30x |
| 손익분기 | **월 2건** ($99 기준) |
| Payback | 첫 거래에서 즉시 회수 |

> LLM API 비용은 연 80% 하락 추세이며, 마진은 지속 개선.

### 3-2. 사업화 추진 전략

**시장 진입 전략:**
- **1차 (Concierge MVP):** 인디해커 커뮤니티 (Product Hunt, Indie Hackers, Show HN)에서 1:1 수동 세팅으로 첫 고객 확보
- **2차 (셀프서브 전환):** 자동화된 웹 대시보드로 전환, URL 입력 → 리포트 자동 생성
- **3차 (구독 + 엔터프라이즈):** 월간 구독 플랜 + 대기업 QA팀 대상 Enterprise 플랜

**포지셔닝:** "인간 10명에 $5,000 vs AI 200명에 $99" — Pre-launch MVP를 위한 가장 빠르고 저렴한 UX 검증

**매출 추정:**

|---|---|---|---|
| 단계 | Concierge MVP | 셀프서브 전환 | 구독 + 엔터프라이즈 |
| 월 신규 고객 | 5~10 | 20~40 | 50~100 |
| **연 매출 (ARR)** | **$18K~36K** | **$150K~300K** | **$500K~1M** |

## 4. 팀 구성 (Team)

### 4-1. 대표자(팀) 구성 보유역량

| 역할 | 주요 역량 | 본 사업 기여 |
|---|---|---|
| **대표 / 엔지니어링 + 리서치** | LLM 통합, Playwright 브라우저 자동화, 감정/인지 모델링, 풀스택 개발 | 핵심 엔진 단독 개발. 감정 파이프라인 설계, Ablation 실험, G2 벤치마크 비교까지 1인 수행 |
| **[채용예정] 프론트엔드** | React, 데이터 시각화 | 셀프서브 대시보드, 리포트 UI, 라이브 뷰 구현 |
| **[채용예정] 그로스** | 커뮤니티 마케팅, 콘텐츠 | Product Hunt 런칭, 인디해커 커뮤니티 운영 |

**핵심 역량:** 대표가 직접 AI 합성 사용자의 감정 모델을 설계하고, Ablation 실험으로 효과를 증명하고, G2 실제 리뷰와 비교 검증까지 수행한 연구-개발 일체형 역량. "AI가 제품을 실제로 사용하게 만드는" 기술적 난이도가 높은 영역에서 작동하는 프로토타입을 이미 확보.

**해자(Moat):** 개별 기술(Playwright, LLM)은 공개 기술이지만, "실제 제품을 사용하면서 감정/인지/의사결정을 시뮬레이션하는" 조합은 어떤 상용 서비스에도 없음. 시뮬레이션 데이터가 쌓일수록 업종별·세그먼트별 벤치마크가 축적되며, 이것이 3년 후의 진정한 해자.

### 4-2. 중장기 사회적 가치 도입계획

- **제품 품질 민주화:** 대기업만 접근 가능했던 UX 리서치($46,000/년)를 $49~199로 내려, 1인 창업자와 소규모 팀도 출시 전 품질 검증이 가능한 환경 조성
- **실패 비용 절감:** Pre-launch 단계에서 치명적 UX 문제를 사전 발견하여, 출시 후 리소스 낭비와 피봇 비용 감소. 스타트업 생존율 향상에 기여
- **다양성 포용 UX:** 비영어권·고령층·장애인 등 기존 테스트에서 소외되는 사용자 세그먼트를 AI 페르소나로 시뮬레이션하여, 더 포용적인 제품 설계 지원
- **일자리 창출:** 사업 성장에 따른 직접 고용 (2027년 3명 → 2028년 8명 목표)
