# VC Handout 스토리텔링 리뷰

> 대상 문서: `docs/vc-meeting-handout.md`
> 리뷰 관점: 스타트업 피칭 코치 — 첫 인상, 시뮬레이션 로그 임팩트, 스토리 흐름, 문장력

## 1. 오프닝 30초 평가: "와, 이거 뭐지?" 유발력

### 현재 상태

```
# Personica — "만드는 건 끝났어. 이거 사람들이 원할까?"
AI 페르소나가 제품을 직접 써보고, 솔직하게 말해줍니다.
```

### 진단: 5/10 — 나쁘지 않지만 "와" 소리가 안 난다

**문제점:**

- **타이틀이 고객의 고통이지, 투자자의 호기심이 아니다.** VC는 "이거 사람들이 원할까?"를 자기 문제로 느끼지 않는다. VC가 느끼는 건 "이 팀이 큰 시장을 먹을 수 있는가?"이다.
- **서브타이틀이 기능 설명이다.** "AI 페르소나가 직접 써보고 솔직하게 말해줍니다"는 What을 말하지만 **Why it matters**를 말하지 않는다.
- **숫자가 없다.** 첫 문장에 규모감이나 텐션을 주는 숫자가 없으면 VC의 눈이 멈추지 않는다.
- **경쟁사 $1.5B 투자 사실이 6번째 섹션에 묻혀있다.** 이건 오프닝 재료다.

### 제안: 오프닝 리라이트

**옵션 A — 시장 텐션 선행:**
```
# Personica

$1.5B가 투입된 "AI 합성 사용자" 시장에서,
아직 아무도 해결하지 못한 한 가지:
**AI가 실제로 제품을 써보는 것.**

Simile($100M), Aaru($50M), Blok($7.5M) — 전부 설문, Figma, 기존 데이터 기반.
Personica는 AI 200명이 실제 브라우저에서 클릭하고, 좌절하고, 이탈합니다.
```

**옵션 B — 시뮬레이션 결과 선행 (쇼크 오프닝):**
```
# Personica

52세 교사 박서연은 Step 7에서 포기했습니다.
"휴지통 아이콘 누르면 다 사라질 것 같아서 무섭다."

31세 PM 김도현은 Step 12까지 갔습니다.
"조건부 로직이 드롭다운 3개가 전부잖아."

같은 제품, 이탈 시점 5스텝 차이, 이유는 완전히 다릅니다.
이 두 사람은 실제 인간이 아닙니다. AI 페르소나입니다.
```

**옵션 B가 강력 추천.** VC가 자료를 넘길 때 첫 3줄에서 "이게 뭐지?" 하게 만들어야 한다. 추상적 선언보다 구체적 장면이 100배 강하다.

## 2. 시뮬레이션 로그 (박서연/김도현) 임팩트 평가

### 현재 상태: 7/10 — 좋지만 최대치가 아니다

**잘된 점:**
- 두 페르소나의 대비가 선명하다 (감정적 이탈 vs 분석적 이탈)
- 직접 인용이 있어서 생생하다
- "이탈 시점 5스텝 차이"라는 정량 포인트가 있다

**아쉬운 점:**

1. **로그 형식이 산문과 인용의 혼합이라 시각적으로 어수선하다.** 타임라인 형태로 바꾸면 "실제 시뮬레이션 로그"의 느낌이 살아난다.

2. **박서연의 감정 궤적이 더 드라마틱해질 수 있다.** 현재는 "당황 → 무서움 → 이탈"인데, 중간에 **한 번 희망을 느꼈다가 좌절하는 순간**이 있으면 더 인간적이다 (이미 데이터가 있다면).

3. **"이것이 세그먼트별 UX 인사이트입니다"라는 마무리가 약하다.** 설명하지 말고 보여주고 끝내야 한다. 독자가 스스로 "이건 다른 건데?"라고 느끼게 해야지, 필자가 "이게 인사이트입니다"라고 라벨링하면 임팩트가 반감된다.

### 제안: 로그 리라이트

```
### 시뮬레이션 로그: Tally.so 폼 빌더

**박서연 — 52세 교사, 기술 불안 성향**
Step 1   랜딩 페이지 도착. "화면이 전부 영어라서 당황스럽다."
Step 3   템플릿 갤러리에서 가장 익숙한 형태 선택 — 안전한 경로 탐색
Step 5   에디터 진입. 블록 삭제 버튼에 커서를 올렸다 뺌
Step 7   "휴지통 아이콘 누르면 다 사라질 것 같아서 무섭다."
         ❌ 자발적 이탈. "구글 폼으로 돌아가겠다."

**김도현 — 31세 PM, 분석형 성격**
Step 1   랜딩 페이지 스킵 → 에디터 직행
Step 4   블록 메뉴 전체 탐색. 드래그앤드롭 테스트
Step 8   조건부 로직 메뉴 진입
Step 12  "조건부 로직이 드롭다운 3개가 전부잖아. Typeform은 중첩이 됐는데."
         ❌ 기능 부족 판단으로 이탈.

같은 제품. 이탈 시점 5스텝 차이. 이유는 완전히 다름.
```

핵심 변경: (1) 스텝 번호를 타임라인처럼 배치해 "실제 로그" 느낌 강화, (2) 마지막 줄에서 "이것이 인사이트입니다" 제거 — 팩트만 놓고 독자가 느끼게 함.

## 3. 전체 스토리 흐름 평가

### 현재 구조

```
1. 문제 (출시 전 검증의 빈자리)
2. 해결 (AI가 실제로 사용) + 시뮬레이션 로그
3. 고객 경험 (5가지 기능)
4. ChatGPT 차별화 + Ablation 결과
5. G2 리뷰 방향 일치
6. 경쟁 환경
7. 유닛 이코노믹스
8. 비치헤드 시장
9. 증명된 것 / 검증 중인 것
10. 팀 & 비전
```

### 진단: 6.5/10 — 논리는 맞지만 순서에 개선 여지

**잘된 점:**
- "문제 → 해결 → 증거" 기본 골격이 있다
- "증명된 것 / 아직 검증 중인 것"의 정직한 분리가 신뢰를 준다
- Ablation 결과를 넣은 건 기술적 깊이를 보여준다

**구조적 문제:**

1. **"고객 경험" 섹션(5가지 기능)이 너무 이르다.** 아직 VC가 "이게 진짜 되는 건지"를 확신하기 전에 기능 목록을 보여주면 "그래서 이게 되긴 하냐"는 의문만 커진다. 증거를 먼저 다 깔고 나서 기능을 보여줘야 한다.

2. **경쟁사 분석이 6번째에 있으면 늦다.** VC가 "왜 이걸 아무도 안 했지?"를 궁금해하는 타이밍은 해결책을 들은 직후다. 시뮬레이션 로그 바로 뒤에 "왜 경쟁사는 못 하는가"를 넣어야 한다.

3. **유닛 이코노믹스와 비치헤드 시장의 순서가 역전이다.** "누구에게 파는가(비치헤드)" → "얼마에 파는가(유닛 이코노믹스)"가 자연스럽다. 시장 크기를 모르는 상태에서 마진 테이블은 의미가 약하다.

4. **ChatGPT 차별화와 G2 비교가 분리되어 있어 "증거"가 흩어진다.** 하나의 "증거" 블록으로 합쳐야 한다.

### 제안: 최적 구조

```
1. 오프닝 — 시뮬레이션 로그로 쇼크 (박서연/김도현)
2. 문제 — 출시 전 검증의 빈자리
3. 해결 — AI가 실제로 사용한다 (How it works 간결하게)
4. 왜 다른가 — 경쟁사 $1.5B 투자했지만 아무도 실제 사용 안 함 + ChatGPT 차별화
5. 증거 묶음 — Ablation 결과 + G2 방향 일치 + 증명된 것/검증 중인 것
6. 시장 — 비치헤드 + TAM/SAM
7. 비즈니스 — 유닛 이코노믹스 + 3년 시나리오
8. 고객 경험 — 라이브뷰/리포트/인터뷰 (여기서야 "그래서 뭘 받는데?"에 답변)
9. 팀 & 비전 + 다음 4주
```

핵심 원칙: **"증거가 충분히 쌓인 뒤에 기능을 보여라."** 기능을 먼저 보여주면 브로셔, 증거를 먼저 보여주면 투자 자료.

## 4. 문장 단위 개선 제안

### 삭제 추천

| 위치 | 현재 문장 | 이유 |
|---|---|---|
| 문제 섹션 | "2026년 기준 새 앱의 75%가 노코드/로코드로 만들어집니다" | 출처가 없으면 VC가 의심한다. 출처를 넣든지, 빼든지 |
| 해결 섹션 | "성격 모델(Big Five)에 기반한 수학적 매핑으로" | 첫 대면 자료에서 Big Five를 설명하면 학술 발표 같아진다. 딥다이브에서 꺼낼 카드 |
| 고객 경험 | "5. 커뮤니티 시뮬레이션" | 아직 구현 안 된 기능을 핸드아웃에 넣으면 "이거 다 안 된 거 아니야?"라는 인상. 현재 동작하는 것만 넣을 것 |
| 마지막 줄 | "200명의 AI가 당신 제품을 써보고 솔직히 말해준다" | "200명"이 앞에서 "100~500명"으로 나왔다가 여기서 200명으로 고정되면 일관성이 깨진다 |

### 강화 추천

| 위치 | 현재 | 제안 |
|---|---|---|
| 문제 섹션 인용 | *"80명에게 개인적으로 공유했는데, 겨우 1명이 가입했다."* | **"80명에게 공유, 1명 가입. 나머지 79명은 왜 안 왔는지 물어볼 방법도 없었다."** — 뒤의 "물어볼 방법이 없다"가 Personica의 가치 제안으로 연결됨 |
| 해결 섹션 첫 줄 | "Personica는 AI 페르소나 100~500명이 실제 브라우저에서 제품을 클릭하고, 입력하고, 탐색합니다." | **"Personica는 AI 200명에게 당신의 제품을 48시간 동안 써보게 합니다. 실제 브라우저에서. 실제 클릭으로."** — 구체 숫자 + 기간 + 짧은 문장 반복이 리듬을 만든다 |
| ChatGPT 비교 | "AI는 본질적으로 제품을 칭찬하는 경향이 있습니다(sycophancy)" | **"ChatGPT는 칭찬 기계입니다. 52세 교사인 척 해달라고 하면 '사용하기 편리하네요!'라고 답합니다. 실제 52세 교사는 영어 화면에서 겁을 먹고 떠났는데도."** — sycophancy 학술 용어 대신 구체 장면 |
| 경쟁사 테이블 | "실제 제품 사용? No / No / No / No / Yes" | 테이블 아래에 한 줄 추가: **"$162M이 투자된 4개 회사가 전부 '제품을 안 써본다'는 같은 한계를 공유합니다."** — 숫자 합산이 임팩트를 만든다 |
| 유닛 이코노믹스 | "손익분기: 월 2건 ($99 기준)" | **"월 2건이면 흑자입니다."** — 표에서 이미 설명했으므로, 핵심 메시지를 짧고 강하게 |
| 비전 마지막 | "시뮬레이션 결과가 축적될수록 업종별/세그먼트별 벤치마크 데이터가 쌓이며, 이것이 3년 후의 진정한 해자가 됩니다." | **"시뮬레이션 1,000회가 쌓이면, 업종별 '정상 이탈률'이 보입니다. 10,000회가 쌓이면, 출시 전에 성공 확률을 예측할 수 있습니다. 데이터가 해자입니다."** — 추상적 선언 대신 구체 스케일 |

### 톤 조정

- **전반적으로 "~합니다" 경어체가 고정되어 있다.** 핸드아웃은 "읽는 문서"이므로 경어체가 맞지만, 시뮬레이션 로그와 핵심 펀치라인에서는 **반말/단문 혼합**이 텐션을 만든다. 예: "같은 제품. 이탈 시점 5스텝 차이. 이유는 완전히 다르다."
- **"핵심은 ~입니다" 패턴이 3회 반복된다.** 필자가 "이게 핵심이야"라고 말하면 할수록 독자는 덜 느낀다. 한 번만 쓰고, 나머지는 보여주기(showing)로 대체할 것.

## 5. 종합 점수 및 우선순위

| 항목 | 점수 | 코멘트 |
|---|---|---|
| 오프닝 30초 | 5/10 | 추상적 선언. 구체 장면이나 숫자로 교체 필요 |
| 시뮬레이션 로그 임팩트 | 7/10 | 대비는 좋으나 형식과 마무리 개선 여지 |
| 스토리 흐름 | 6.5/10 | 기능이 너무 앞, 경쟁사가 너무 뒤, 증거가 흩어짐 |
| 문장력 | 6/10 | 학술적 톤이 군데군데 섞여 있고, 펀치라인 부족 |
| **종합** | **6/10** | 내용은 충분. 순서와 톤 재배치만으로 8+/10 가능 |

### Top 3 즉시 실행 (Impact 순)

1. **오프닝을 시뮬레이션 로그로 교체** — 30초 안에 "이건 다르다"를 느끼게 하는 가장 확실한 방법
2. **섹션 순서 재배치** — 오프닝(로그) → 문제 → 해결 → 경쟁사(왜 아무도 못 했나) → 증거 묶음 → 시장 → 비즈니스 → 기능 → 팀
3. **"커뮤니티 시뮬레이션" 섹션 삭제** — 아직 없는 기능은 신뢰를 깎는다. 있는 것만 강하게 보여줄 것
