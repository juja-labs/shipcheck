## 1. 과장되거나 증명 없이 주장하는 부분

### 1.1 [심각] "AI 페르소나 100~500명이 실제 브라우저에서 제품을 클릭하고, 입력하고, 탐색합니다"

**핸드아웃 주장** (라인 26): 100~500명 규모의 병렬 시뮬레이션이 마치 현재 가능한 것처럼 서술됨.

**실제 상태**:
- 코드에서 실험이 실행된 페르소나는 **2명** (p001 박서연, p002 김도현). 각각 10스텝, 9스텝.
- `run_experiment.py`의 기본값은 5명 동시 실행 (`DEFAULT_PERSONAS = ["b018", "b005", "b001", "b023", "b019"]`)
- 50명 이상 동시 실행에 대한 인프라, 비용 검증, 안정성 테스트가 전혀 없음
- discovery-plan.md 자체가 "50명 x 3개 제품 정량 실험"을 **아직 미실행**으로 명시

**판정**: 현재 시점에서 "100~500명"은 **비전이지 현실이 아님**. 2명 심층 테스트를 마치 대규모 시뮬레이션이 돌아가는 것처럼 포장하면 VC에게 신뢰를 잃을 수 있음.

### 1.2 [심각] Ablation 실험 결과의 과장 표현

**핸드아웃 주장** (라인 73~81): "실제 Ablation(대조 실험) 결과"라고 표기하며 박서연의 PU 0.3~0.5 vs 0.6~0.8, "포기하고 싶다" 4회 vs 0회, 매몰 비용 심리 발생 등을 제시.

**실제 상태**:
- 코드베이스에 ablation 실험 코드가 없음. `run_experiment.py`에 ablation 모드가 없음
- 감정 파이프라인을 끈 상태에서 동일 페르소나를 돌리는 비교 실험의 코드/로그가 존재하지 않음
- "실제 Ablation 결과"라는 표현은 **체계적인 대조 실험**을 암시하지만, n=1 (한 명의 페르소나)에 대한 비교이고, 그마저도 재현 가능한 코드가 없음

**판정**: "Ablation" 표현이 학술적 신뢰를 빌려오려는 의도처럼 보이지만, 실제로는 **비공식 관찰 수준**. n=1이라는 점을 명시해야 함.

### 1.3 [중간] G2 리뷰 88개와의 "방향 일치" 주장

**핸드아웃 주장** (라인 87~95): G2 리뷰 88개를 크롤링하여 시뮬레이션 결과와 비교. 방향 일치.

**실제 상태**:
- `src/review/benchmark.py`에 비교 파이프라인이 구현되어 있음 (키워드 기반 테마 추출, Jaccard 유사도, Pearson 상관). 이 부분은 코드가 존재하여 실행 가능한 상태
- 그러나 비교 방법이 **키워드 매칭 기반**(THEME_KEYWORDS 딕셔너리)이라 매우 조잡함. 임베딩/의미 유사도가 아닌 문자열 포함 여부로 판정
- "방향 일치"의 정량 지표(Jaccard 값, Pearson r 값)가 핸드아웃에 없음. 구체적 수치 없이 "방향이 일치"라고만 표기
- 합성 리뷰는 **2명**에 대해서만 생성됨 (`runs/experiment/reviews/` — p001, p002)
- G2 크롤러는 존재하나 (`src/data/crawlers/g2_crawler.py`), 88개 vs 2개를 비교한 것임

**판정**: 88개 G2 리뷰 대비 2개 합성 리뷰로 "방향 일치"를 주장하는 것은 통계적으로 무의미. n=2에서 "테마가 비슷해 보인다"는 체리피킹에 가까움.

### 1.4 [중간] "수학적 감정 모델로 이 문제를 해결합니다"

**핸드아웃 주장** (라인 71): Sycophancy 문제를 수학적 감정 모델로 "해결"했다고 서술.

**실제 상태**:
- `src/layer3_emotion/engine.py`에 OCC -> PAD 매핑, SDE noise, decay, 이탈 판정이 구현되어 있음. 이 부분은 실제로 작동하는 코드
- 그러나 이 엔진은 PAD 수치를 계산해서 LLM 프롬프트에 전달하는 방식. 최종 행동/발화 결정은 여전히 **LLM이 함** (`prompt_builder.py`의 시스템 프롬프트로 제약)
- LLM이 PAD 수치를 "참고"할 뿐, 수학적으로 행동이 결정되는 것은 이탈 판정(`check_abandonment`)뿐
- sycophancy를 "해결"했다는 표현은 과장. "완화(mitigate)"가 정확

**판정**: 감정 엔진이 존재하고 작동하는 것은 사실이나, "해결"은 과장. LLM의 최종 출력을 수학적으로 강제(hard constraint)하는 것은 이탈 판정뿐이고, 나머지는 soft constraint(프롬프트 지시).

### 1.5 [중간] 유닛 이코노믹스의 비현실적 가정

**핸드아웃 주장** (라인 126~144): 50명/$49, 200명/$99 등의 가격과 COGS.

**문제점**:
- LLM API 비용 계산의 기반이 불명확. 실제 실험에서 2명 실행 비용이 `run_experiment.py`에 기록되어야 하나, 이를 바탕으로 한 역산(back-calculation)이 제시되지 않음
- `run_experiment.py`가 Claude CLI를 subprocess로 호출하는 구조인데, Claude Sonnet 기준 50명 x 25스텝 x 매 스텝 LLM 콜 = **최소 1,250회 LLM 호출**. 이를 $2.38(GPT-4.1 mini 기준)이라 하려면 호출당 ~$0.002 수준이어야 하는데, 현재 Claude Sonnet을 쓰고 있으므로 비용이 크게 다름
- CAC $15~25는 커뮤니티 기반 가정인데, 구체적 채널 전략이 없음
- LTV/CAC 12~30x는 반복 구매를 전제하는데, 반복 사용률 자체가 "아직 검증 중"으로 분류됨

**판정**: 유닛 이코노믹스는 **모델이 GPT-4.1 mini로 전환된다는 가정** 하에 계산된 낙관적 시나리오. 현재 Claude Sonnet 기준 비용은 이보다 높을 것. 가정을 명시해야 함.

### 1.6 [경미] "5-Layer 파이프라인"의 구현 상태 과장

**핸드아웃 주장** (라인 28): "성격 모델(Big Five)에 기반한 수학적 매핑"이 마치 전 레이어가 작동하는 것처럼 서술.

**실제 구현 상태**:
| Layer | 설계 문서 | 코드 구현 | 실험 적용 |
|-------|----------|----------|----------|
| Layer 1: Persona Profile | 완전 | 완전 (`models.py`, `generator.py`) | 적용됨 |
| Layer 2: Cognitive | 완전 | **부분적** (`compute_cognitive_load` 하나만 구현. Information Foraging, Mental Model Matching, TAM은 미구현) | 인지 부하만 |
| Layer 3: Emotion | 완전 | **핵심 구현** (`engine.py` — OCC, PAD, SDE, decay, 이탈) | 적용됨 |
| Layer 4: Decision | 완전 | **미구현** (BDI-E, Fogg, Satisficing 전부 코드 없음. LLM 프롬프트에 자연어로만 포함) | 프롬프트만 |
| Layer 5: Memory | 완전 | **미구현** (Memory Stream, Reflection, Habit Strength 전부 코드 없음) | 없음 |

**판정**: 5-Layer 중 코드로 실제 작동하는 것은 **Layer 1과 Layer 3의 핵심부분**. Layer 2는 부분적, Layer 4/5는 설계만 존재. "5-Layer 파이프라인 실행"이라는 표현은 과장.

### 1.7 [경미] 시장 규모 주장 근거 부재

**핸드아웃 주장** (라인 101): "$1.5B+가 투자된 시장"

**문제점**: Simile $100M + Aaru $50M+ + Blok $7.5M + Artificial Societies $5.35M = ~$163M. 나머지 $1.34B의 출처가 없음. 10배 가까이 부풀려짐.

## 2. "증명된 것"과 "아직 검증 중인 것" 섹션 솔직성 평가

### 전반적 평가: **방향은 솔직하나, "증명된 것"의 기준이 너무 낮음**

#### "증명된 것"으로 분류된 항목별 검증

| 항목 | 핸드아웃 주장 | 실제 근거 수준 | 솔직성 판정 |
|------|-------------|--------------|-----------|
| 페르소나 분화 | "유의미하게 다름 (2명 심층 + Tally 테스트)" | 2명(p001, p002)이 다른 행동을 보인 것은 사실. 10스텝/9스텝의 로그 존재. 그러나 n=2에서 "통계적 유의미"는 불가. | **"관찰됨"이 맞지 "증명됨"은 과장** |
| 감정 파이프라인 효과 | "Ablation으로 입증" | 위 1.2 참조. 재현 가능한 ablation 코드 없음 | **"증명됨"은 과장, "초기 관찰"이 정확** |
| G2 방향 일치 | "주요 테마 방향적으로 일치" | 비교 코드는 존재하나 n=2 합성 리뷰 vs 88 G2 리뷰. 정량 지표 미공개 | **"시사점 있음" 수준, "증명"은 아님** |
| 원가 구조 | "실제 API 가격 기반 COGS 검증" | GPT-4.1 mini 기준 이론 계산. 현재 Claude Sonnet 사용 중 | **가정 조건부로 맞지만, 현재 스택과 불일치** |

#### "아직 검증 중인 것"의 솔직성

이 섹션은 비교적 솔직함:
- 대규모 통계적 분화 미실행 인정
- 고객 지불 의향 미검증 인정
- 리포트 활용성 미검증 인정
- 다중 세션 미검증 인정

**그러나 빠진 것이 있음**:
1. **Layer 4/5 미구현**: "아직 검증 중"이 아니라 **아직 만들지도 않은 것**. 이 사실이 어디에도 언급 안 됨
2. **Playwright 범용성 미검증**: 임의의 웹 제품에서 안정적으로 동작하는지 (SPA, 복잡한 인증 등) 검증 없음
3. **비용 현실성**: 현재 Claude Sonnet 기준 실제 비용이 얼마인지 측정하지 않음

**제안**: "증명된 것" → "관찰된 것(초기 신호)"으로 표현 수정. n=2라는 사실을 명시. "아직 검증 중" 섹션에 구현 상태도 포함.

## 3. 경쟁사 비교 테이블 공정성 평가

### 비교 테이블 (라인 103~111)

| 점검 항목 | 평가 |
|----------|------|
| Simile: "No" | **공정**. Simile은 행동 예측 모델이지 실제 제품 사용이 아님 |
| Aaru: "No" | **공정**. Aaru는 설문/여론 시뮬레이션 |
| Blok: "No" | **대체로 공정하지만 뉘앙스 누락**. Blok은 Figma 프로토타입 기반 인지 시뮬레이션을 하므로 "제품을 시각적으로 체험"은 함. "실제 웹 제품 조작"은 안 하는 게 맞음 |
| Artificial Societies: "No" | **공정**. 네트워크 시뮬레이션 |
| UXAgent (Amazon): **테이블에서 누락** | **부당한 생략**. UXAgent는 실제로 Playwright로 웹 제품을 사용하는 오픈소스. discovery-plan.md에는 "부분적 (쇼핑 전용, 감정 없음)"으로 포함되어 있는데 핸드아웃에서 삭제됨. 가장 직접적인 기술 경쟁자를 숨기는 것은 불공정 |

### "실제 제품을 AI가 사람처럼 사용하는 상용 서비스는 Personica뿐" 주장

**문제**:
1. UXAgent(Amazon, CHI 2025)가 정확히 동일한 접근을 함. 오픈소스이고 상용은 아니지만, Amazon 내부에서 사용 중. 핸드아웃의 discovery-plan에는 있으나 경쟁사 테이블에서 제외
2. "상용 서비스"라는 한정을 붙여 회피하고 있으나, Personica 자체도 아직 상용이 아님 (고객 0, 매출 0)
3. "경쟁사 전원이 이 접근을 회피함" → "회피"가 아니라 **"다른 전략을 선택함"**이 중립적 표현. 기술적 난이도/비용/범용성 문제로 다른 접근을 택한 것을 "회피"로 프레이밍하면 경쟁사가 무능한 것처럼 보임

**제안**: UXAgent를 테이블에 포함하되 차별점(감정 시뮬레이션 없음, 쇼핑 전용, 범용 아님) 명시. "회피" → "선택하지 않음"으로 수정.

## 4. 기술 해자(Moat) 주장 현실성 평가

### 핸드아웃의 해자 주장 (라인 203)

> "개별 기술(Playwright, LLM)은 공개 기술이지만, '실제 제품을 사용하면서 감정/인지/의사결정을 시뮬레이션하는' 조합은 어떤 상용 서비스에도 없습니다. 시뮬레이션 결과가 축적될수록 업종별/세그먼트별 벤치마크 데이터가 쌓이며, 이것이 3년 후의 진정한 해자가 됩니다."

### 현실성 평가

**단기 해자 (현재~1년): 거의 없음**
- Playwright + LLM 조합은 재현에 6~8주면 충분. 코드베이스 자체가 크지 않음 (핵심 로직 ~1000줄)
- OCC->PAD 매핑, SDE noise, Big Five -> 행동 파라미터 매핑은 simulation-engine.md에 완전히 문서화되어 있어, 이 문서를 보고 재구현 가능
- 실제 구현된 Layer 3 감정 엔진이 ~185줄, Layer 1 모델이 ~335줄. 복잡도가 높지 않음

**중기 해자 (1~3년): 잠재적이지만 조건부**
- "벤치마크 데이터 축적"은 유효한 해자 후보이나, **고객이 없으면 축적도 없음**
- 축적 속도가 의미 있으려면 **월 100건+ 시뮬레이션**이 필요. 현재 Y1 목표가 월 5~10건이므로 데이터 해자가 의미 있어지려면 최소 2~3년 필요
- 그 사이에 Simile($100M), Aaru($50M+)가 동일 접근을 추가할 가능성 높음. 자금력 차이가 압도적

**장기 해자 (3년+): 불확실**
- 핸드아웃이 주장하는 "진정한 해자"까지 도달하려면: (a) 시뮬레이션 충실도 증명, (b) 고객 확보, (c) 데이터 축적 3단계 모두 성공해야 함
- 현재 (a)도 미증명 상태

**현실적 해자 평가**:
- 기술 해자: 약함 (재현 가능)
- 실행 해자(time-to-market): 약간 있음. "Playwright로 실제 제품 사용 + 감정 시뮬레이션"을 먼저 시도한 것 자체가 가치. 하지만 선행 기간이 매우 짧음
- 데이터 해자: 아직 없음 (고객 0)
- IP 해자: 약함. 논문 기반 공개 방법론 조합

## 5. 수정 제안

### 5.1 즉시 수정 (정직성/신뢰 관련)

## 요약: CTO 관점 종합 판단

| 영역 | 강점 | 약점 |
|------|------|------|
| 기술 설계 | 5-Layer 아키텍처 설계가 학술적으로 탄탄. 논문 레퍼런스 30+편 기반. 감정 엔진(Layer 3) 구현이 실제로 작동하며, 결정론적 이탈 판정은 sycophancy 완화에 효과적인 접근 | 설계와 구현 사이 갭이 큼. 5개 레이어 중 2.5개만 코드화. Layer 4 Decision은 전적으로 LLM 프롬프트에 의존 |
| 증거 수준 | 2명 심층 테스트에서 관찰된 분화와 감정 궤적이 그럴듯함. 합성 리뷰(p001 박서연)의 품질이 실제 G2 리뷰와 톤/구조가 유사 | n=2로 "증명"을 주장하는 것은 학술적으로도, 비즈니스적으로도 위험. 50명 실험이 최소 필요 |
| 경쟁 포지셔닝 | "실제 브라우저 조작"이라는 포지션은 차별화가 명확하고, 경쟁사들이 이 접근을 안 하는 건 사실 | UXAgent를 의도적으로 생략한 것은 신뢰를 깎음. "$1.5B" 시장 규모 부풀림 |
| 유닛 이코노믹스 | Gross Margin 78%+ 구조는 GPT-4.1 mini 기준으로 현실적 | 현재 스택(Claude Sonnet)과 불일치. LTV/CAC는 반복 사용 미검증 상태에서 무의미 |

**핵심 메시지**: 기술 비전과 설계는 인상적이지만, 핸드아웃이 "비전"과 "현실"의 경계를 흐리고 있음. n=2 관찰을 "증명"으로, 미구현 레이어를 "파이프라인 실행"으로, 직접 경쟁자를 숨기는 것은 기술 실사에서 바로 잡힐 문제. VC 미팅에서 이런 부분이 드러나면 전체 신뢰가 무너질 수 있으므로, **솔직하게 현재 상태를 인정하면서도 비전과 실행 계획의 명확성으로 승부하는 것이 더 강한 전략**.
