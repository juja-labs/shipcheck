## 1. 매출 구조: 건별 과금 (복잡도별 구간)

### 기본 모델

```
매출 = 건수 × 건당 가격
```

| 티어 | 페르소나 수 | 스텝 | 건당 가격 | 타겟 세그먼트 | 용도 |
|------|:---------:|:---:|:--------:|-------------|------|
| **Starter** | 10명 | 30 | **$49** | 인디해커 첫 시도, 아이디어 검증 | 랜딩·온보딩 검증 |
| **Growth** | 30명 | 100 | **$149** | 인디해커 본격, PM 탐색 | 핵심 기능 평가 |
| **Scale** | 100명 | 200 | **$499** | PM/팀, 릴리즈 전 검증 | 전체 UX + 세그먼트 분석 |

> 스텝 수로 제품 복잡도에 따라 자연스럽게 가격 차등.
> 고객이 자기 제품에 맞는 플랜을 직관적으로 선택 가능.

### 반복 사용 패턴 확인 시 → 추가 상품 설계 (2단계)

현재 SOT 기준 매출 모델은 **건별 과금만 적용**한다.
반복 사용 패턴과 팀 협업 수요가 확인되면, 구독/크레딧 구조는 별도 문서에서 재설계한다.

## 3. 시기별 매출 추정

### 핵심 가정

| 가정 | 값 | 근거 |
|------|:--:|------|
| 1차 진입시장 규모 | 3,425~5,540명/년 | PH + Show HN + BetaList + PM + AppSumo |
| 도달률 (1년차) | 5~10% | 커뮤니티 + PLG 채널 |
| 유료 전환율 | 10~15% | 프리미엄 SaaS 평균 |
| 연간 사용 횟수 | 2~3회 (인디), 6~12회 (PM) | 초기 가격 가설 기반 |
| 혼합 평균 연매출 | 1년차 $600, 3년차 $2,000, 5년차 $3,000 | 세그먼트 구성 변화 |

### 추정 결과

| 시점 | 유료 고객/건수 | 기준 단가 또는 연간 객단가 | **매출** | 주요 이벤트 |
|------|:------------:|:-----------------------:|:------:|-----------|
| **제품 개발 기간** (5개월) | 10~20건 (누적) | $149/건 | **$1K~$3K** | 알파→초기 유료 검증 |
| **1년차** | 200~400개사 | $600/년 | **$120K~$240K** | PH 런칭, 반복 사용 확인 |
| **2년차** | 500~1,000개사 | $1,200/년 | **$600K~$1.2M** | 모바일 확장, UX리서치 진입 |
| **3년차** | 800~1,500개사 | $2,000/년 | **$1.6M~$3M** | SMB 팀 확장 + 미드마켓 |
| **5년차** | 1,000~2,000개사 | $3,000/년 | **$3M~$6M** | 엔터프라이즈 Pull |

### SOM과의 정합 확인

```
SOM (보수적): SAM $0.9B × 0.5% = $4.6M
SOM (기준):   SAM $1.7B × 1.0% = $16.5M

5년차 매출 추정: $3M~$6M

→ 보수적 SOM $4.6M과 5년차 $6M이 정합
→ 기준 SOM $16.5M은 7~10년차 또는 엔터프라이즈 가속 필요
→ 사업계획서에는 "5년 SOM $3M~$6M, 장기 $4.6M+" 으로 서술
```
