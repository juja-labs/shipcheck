# Unit Economics — Personica

**Date**: 2026-03-22
**기준**: PAYG (Pay-As-You-Go) 건별 과금, 제품 복잡도별 티어
**모델 전략**: 경량 모델 (60%) + 고성능 추론 모델 (40%) 혼합
**원가 검증**: 30명 페르소나 시뮬레이션 검증 완료

## 1. 모델 전략 & 원가 구조

### 1.1 경량 6 : 고성능 추론 4 혼합

5-Layer 엔진의 레이어별로 최적 모델 배정:

| 레이어 | 역할 | 모델 | 근거 |
|--------|------|------|------|
| Perceive (DOM 파싱) | 페이지 → 관찰 | **경량** | 구조화된 파싱, 창의성 불필요 |
| Act (Playwright) | 브라우저 명령 | **경량** | 단순 명령 변환 |
| Memory/Reflection | 기억 저장·회상 | **경량** | 패턴 매칭 + 요약 |
| Emotion (OCC-PAD-SDE) | 감정 평가·진화 | **고성능 추론** | 감정 뉘앙스가 핵심 |
| Decision (BDI-E) | 행동 결정 | **고성능 추론** | 동기·갈등·포기 시뮬레이션 |

토큰 비중: Perceive(DOM) + Act + Memory ≈ 60%, Emotion + Decision ≈ 40%

### 1.2 혼합 모델 원가

| 구분 | 토큰 비율 | Input 비용 | Output 비용 | 소계 |
|------|:--------:|:---------:|:----------:|:----:|
| 경량 60% | 141K in / 31K out | $0.113 | $0.125 | **$0.24** |
| 고성능 추론 40% | 94K in / 21K out | $0.283 | $0.312 | **$0.60** |
| **LLM 합계** | | | | **$0.84** |

> 30명 시뮬레이션 검증 기반 토큰 사용량 산출.

## 2. 제품 복잡도별 원가

대상 제품의 복잡도에 따라 필요 스텝 수가 달라지고, 원가가 변동한다.

### 2.1 복잡도별 스텝 가이드

| 복잡도 | 예시 제품 | 스텝 수 | 포착 범위 |
|--------|----------|:------:|----------|
| **경량** | 랜딩페이지, 예약 사이트, 설문 폼 | 30스텝 | 첫인상, 온보딩, 진입 장벽 — "시작도 못 하고 떠나나?" |
| **중간** | 일반 웹 서비스, 업무 도구 | 100스텝 | 핵심 기능, 조건부 로직, 설정 — "기능을 제대로 쓸 수 있나?" |
| **복잡** | 엔터프라이즈 솔루션, 복합 플랫폼 | 200스텝 | 고급 설정, 외부 연동, 커스터마이징 — "전문 사용자가 만족하나?" |

> **스텝 기준 근거**: 자체 30명 실험에서 경량 웹 서비스 대상 무제한 탐색 시 평균 73스텝 소진.
> 경량 서비스가 73스텝이므로 핵심 기능 커버에 30스텝이면 충분하며,
> 중간 복잡도는 기능 수 3~5배 → 100스텝, 복합 플랫폼은 200스텝으로 설정.
> UXAgent(Amazon, CHI 2025)도 태스크당 15~50 스텝으로 유의미한 UX 이슈를 포착함.

### 2.2 스텝별 COGS/페르소나 (경량6:고성능4)

경량6:고성능4 혼합 기준, 스텝 수에 비례하여 선형 스케일링:

| 스텝 | LLM/명 | 인프라/명 | **COGS/명** |
|:----:|:------:|:--------:|:----------:|
| 30 | $0.35 | $0.04 | **$0.39** |
| 100 | $1.15 | $0.10 | **$1.25** |
| 200 | $2.30 | $0.19 | **$2.49** |

> Memory Summarization으로 컨텍스트 윈도우를 관리하여 스텝당 비용을 일정하게 유지.

### 2.3 페르소나 수의 통계적 의미

| 페르소나 수 | 세그먼트 커버 | 의미 |
|:---------:|:----------:|------|
| 10명 | 2~3개 | 방향성만 확인 |
| 30명 | 9개 | 통계적 경향 확인 가능 |
| 100명 | 세그먼트별 10명 | 세그먼트 내 일관성까지 검증 |

> **근거**:
> - **자체 30명 실험**: Big Five 성격 모델 기반으로 9개 세그먼트를 자동 생성하여 시뮬레이션.
>   세그먼트 간 유의미한 행동 분화(이탈 지점, 감정 반응, 기능 사용 패턴 차이)를 확인함.
> - **Nielsen (2000)**: 사용성 테스트에서 5명이면 주요 이슈의 80%를 발견. 10명은 방향성 확인에 충분.
> - **UXAgent (Amazon, CHI 2025)**: 5~10명의 합성 페르소나로 실제 사용자 테스트와 유사한 UX 이슈를 포착.

## 3. 가격 체계 (PAYG, 복잡도별 티어)

### 3.1 티어 구성 & 매출총이익

| 티어 | 페르소나 | 스텝 | 가격 | 단가/명 | COGS | **매출총이익** | **마진** | 용도 |
|------|:-------:|:---:|:----:|:------:|:----:|:----------:|:------:|------|
| **Starter** | 10명 | 30 | **$49** | $4.90 | $4.20 | $44.80 | **91%** | 랜딩·온보딩 검증 |
| **Growth** | 30명 | 100 | **$149** | $4.97 | $38.00 | $111.00 | **75%** | 핵심 기능 평가 |
| **Scale** | 100명 | 200 | **$499** | $4.99 | $250.00 | $249.00 | **50%** | 전체 UX + 세그먼트 분석 |

> **Scale 마진 참고**: 50%는 SaaS 평균(70%) 대비 낮으나:
> ① LLM 비용 연 30~50% 하락 → 2년 후 73%+ 전망
> ② Scale 고객은 반복 사용 → 구독 전환 대상
> ③ 엔터프라이즈 가격($599~$699)으로 조정 여지

Personica는 **$49부터 건별 구매**로 인디해커·초기 PM이 접근 가능.

#### ② 인간 리서치 대비 비용

| 방식 | 30명 기준 비용 | Personica Growth 대비 |
|------|:----------:|:-------------------:|
| User Interviews (인터뷰) | $1,230~$2,940 | **8~20배 비쌈** |
| Respondent.io (모더레이티드) | $660~$1,200 | **4~8배 비쌈** |
| UserTesting (사용성 테스트) | $240~$300 + 시트비 | **2~3배 비쌈** |
| 에이전시 리서치 1건 | $15,000~$50,000+ | **100~330배 비쌈** |
| 포커스그룹 1세션 (8~10명) | $4,000~$12,000 | **27~80배 비쌈** |
| **Personica Growth** | **$149** | — |

→ 동일 규모 인간 테스트의 **1/4~1/330 비용**. 인간 리서치를 대체하는 것이 아니라,
탐색 단계에서 가설을 빠르게 검증한 뒤 소수 인간 테스트로 확인하는 **보완재** 포지션.
(가격 출처: 각 플랫폼 공식 가격표 및 G2 리뷰 기준 — competitive-pricing.md 참조)

#### ③ 지불의향 (WTP)

| 세그먼트 | WTP 범위 | 근거 |
|---------|:--------:|------|
| 인디해커/솔로빌더 | $29~99/건 | 경쟁사 Starter 가격대 ($39~99) 및 PLG SaaS 진입 가격 참고 |
| 스타트업 PM | $99~199/건 | 경쟁사 SMB 가격대. Maze SMB 평균 $2,840/년 ≈ $237/월 (competitive-pricing.md) |
| 펀딩 스타트업 | $199~499/건 | UserTesting 엔터프라이즈 ~$148K/년 대비 극소 비용. 리서치 예산 1건 수준 |

→ Starter $49 = 인디해커 WTP 하단, Growth $149 = PM WTP 중간, Scale $499 = 펀딩팀 WTP 상단.

#### ④ 복잡도별 차등 가격

스텝 수로 제품 복잡도에 따라 자연스럽게 가격 차등:
- 고객이 **자기 제품에 맞는 플랜을 직관적으로 선택** 가능
- "내 랜딩페이지는 Starter면 충분하고, SaaS는 Growth가 맞겠다"
- 과금 기준이 명확하여 **사전 비용 예측 가능** (크레딧/종량제 대비 장점)

#### ⑤ 가격 인상 여지

| 현재 | 인상 가능 | 근거 |
|:----:|:--------:|------|
| $4.90/명 | $6~8/명 | Synthetic Users 최저 $2~27 범위 내 |
| Growth $149 | $199~249 | PM WTP $99~199 상단 |
| Scale $499 | $599~699 | 마진 50%→60~65% 개선 |

→ 현재 가격은 **시장 침투 가격**. PMF 확인 후 가치 기반 인상 여지 충분.

## 8. 원가 최적화 전망

현재 원가는 **주류 상용 API를 보수적으로 산정**한 수치이며, 다음 3가지 경로로 마진이 점진적으로 개선된다.

### 8.1 원가 절감 경로

| 경로 | 설명 | 예상 절감 | 시점 |
|------|------|:--------:|:----:|
| **① LLM API 가격 하락** | 업계 전반 연 30~50% 하락 추세. 동일 성능 기준 API 단가가 매년 절반 수준으로 하락 중 (a16z "The Cost of AI Inference", 2024) | 30~50%/년 | 자동 |
| **② 에이전트 캘리브레이션** | 페르소나별 불필요 탐색 스텝 제거, 레이어별 프롬프트 경량화, 행동 패턴 학습을 통한 스텝 효율화 | 15~30% | 6~12개월 |
| **③ 독자/오픈소스 모델 전환** | 경량 레이어(60%)를 독자 파인튜닝 모델 또는 고성능 오픈소스 모델로 전환. 감정·의사결정 레이어(40%)의 품질은 유지 | 40~60% (경량 레이어) | 12~18개월 |

#### 에이전트 캘리브레이션 상세

| 최적화 항목 | 내용 | 효과 |
|-----------|------|------|
| **스텝 효율화** | 반복적 탐색 패턴 감지 → 불필요 스텝 자동 스킵 | 총 스텝 수 10~20% 감소 |
| **프롬프트 경량화** | 레이어별 프롬프트를 시뮬레이션 데이터 기반으로 정제 → 토큰 수 절감 | 턴당 토큰 15~25% 감소 |
| **행동 템플릿 학습** | 빈출 행동 패턴(로그인, 네비게이션 등)을 템플릿화 → LLM 호출 생략 | LLM 호출 수 10~15% 감소 |
| **적응형 모델 라우팅** | 턴 복잡도에 따라 경량↔고성능 모델을 동적 전환 | 고성능 모델 사용 비율 40%→25% |
