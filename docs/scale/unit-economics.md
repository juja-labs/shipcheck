## 1. 시뮬레이션 원가 구조

### 1.1 페르소나 1명당 토큰 사용량 (10스텝 기준)

lean-canvas 기준, 페르소나 1명 × 1스텝 = 4회 LLM 호출:

| 컴포넌트 | 역할 | Input 토큰 | Output 토큰 | LLM 콜 |
|----------|------|-----------|------------|--------|
| Perceive (DOM 파싱) | 현재 페이지 → 관찰 | ~2,000 | ~500 | 1 |
| Emotion Eval (OCC-PAD) | 감정 평가 | ~1,500 | ~300 | 1 |
| Decision (BDI-E) | 다음 행동 결정 | ~2,000 | ~400 | 1 |
| Act (Playwright) | 브라우저 명령 생성 | ~800 | ~200 | 1 |
| **스텝 합계** | | **~6,300** | **~1,400** | **4** |

**페르소나 1명 × 10스텝:**
- Input: 63,000 토큰
- Output: 14,000 토큰
- LLM 콜: 40회

> 스텝 진행에 따라 history/context 누적으로 input 증가. 위 수치는 보수적 평균.

### 2.3 가격 결정 근거

- **WTP**: 인디해커 $29~99 (Van Westendorp), PM $99~199
- **경쟁사 대비**: Synthetic Users $2~27/명 (최소 $10K/년) 대비 진입장벽 압도적 낮음

- **마진**: 어떤 모델에서든 70%+ → SaaS 업계 평균(70~80%) 충족

### 월간 고정비

| 항목 | 초기 | 성장기 |

### 손익분기 계산

```
N = 고정비 / (가격 - COGS)
```

| 티어 | 가격 | COGS (Haiku) | 공헌이익 | BEP (고정비 $55) | BEP (고정비 $150) |
