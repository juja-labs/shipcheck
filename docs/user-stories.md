## Story 1: 검증 프로젝트 생성

**Title:** 검증 프로젝트 생성

**Description:** As a 제품 책임자, I want to 스테이징 URL과 테스트 계정, 타겟 유저 설명을 입력해서 검증 프로젝트를 만들고 싶다, so that Personica가 내 제품을 분석하고 시뮬레이션을 준비할 수 있다.

**Acceptance Criteria:**
1. 스테이징 URL 입력 시 시스템이 접속 가능 여부를 즉시 확인한다.
2. 테스트 계정(ID/PW 또는 매직링크)을 입력할 수 있고, 로그인 불필요 제품은 건너뛸 수 있다.
3. "이 제품의 타겟 유저가 누구인가"를 자연어로 설명할 수 있다. (예: "20~30대 1인 창업자, 노코드 경험 있음")
4. 선택적으로 "특히 검증하고 싶은 시나리오"를 추가할 수 있다. (예: "온보딩 후 첫 프로젝트 생성까지")
5. 선택적으로 시뮬레이션 후 페르소나에게 물어볼 설문 문항을 추가할 수 있다. (기본: SUS 10문항 자동 포함)
6. "분석 시작"을 누르면 프로젝트가 생성되고 자동 분석이 시작된다.
7. 테스트 계정 정보는 암호화 저장되고, 분석 완료 후 자동 삭제 옵션이 제공된다.

## Story 2: 제품 구조 분석 결과 확인

**Title:** 자동 제품 분석 결과 확인

**Description:** As a 제품 책임자, I want to Personica가 분석한 제품 구조(페이지, 기능, 플로우)를 확인하고 보정하고 싶다, so that 시스템이 내 제품을 정확히 이해한 상태에서 시뮬레이션이 진행된다.

**Acceptance Criteria:**
1. 분석 진행 상황을 실시간으로 볼 수 있다. (크롤링 중 → 기능 파악 중 → 플로우 매핑 중)
2. 발견된 페이지 목록, 주요 기능, 유저 플로우가 구조화되어 표시된다.
3. 각 플로우에 "맞다/틀리다/빠졌다" 피드백을 줄 수 있다.
4. 시스템이 놓친 플로우나 페이지를 수동으로 추가할 수 있다.
5. 확인 완료 시 다음 단계(페르소나 생성)로 넘어간다.

## Story 3: 페르소나 리뷰 및 조정

**Title:** AI 생성 페르소나 리뷰 및 조정

**Description:** As a 제품 책임자, I want to 자동 생성된 페르소나 목록을 확인하고 조정하고 싶다, so that 내 타겟에 맞는 다양한 사용자 유형이 빠짐없이 테스트된다.

**Acceptance Criteria:**
1. 타겟 설명 기반으로 페르소나 N명이 카드 형태로 표시된다.
   - 각 카드: 이름, 나이대, 직업, 기술 숙련도, Big Five 요약, JTBD 한 줄
2. 페르소나가 세그먼트별로 그룹핑되어 있다: **코어** (타겟 직접 매칭), **엣지** (인접하지만 고객이 놓칠 수 있는), **와일드카드** (의외의 사용자군).
3. 개별 페르소나를 ON/OFF 토글할 수 있다.
4. 속성(나이대, 기술 숙련도, Big Five 값)을 미세 조정할 수 있다.
5. "이런 유저도 추가해줘"를 자연어로 입력하면 추가 페르소나가 생성된다.
6. 최종 구성 확정 시 시뮬레이션 준비 상태로 전환된다.

## Story 4: 시뮬레이션 실행 및 모니터링

**Title:** 시뮬레이션 실행 및 실시간 모니터링

**Description:** As a 제품 책임자, I want to 페르소나들이 내 제품을 사용하는 과정을 실시간으로 보고 싶다, so that 시뮬레이션이 정상 진행되는지 확인하고 흥미로운 행동을 바로 포착할 수 있다.

**Acceptance Criteria:**
1. "시뮬레이션 시작"을 누르면 전체 페르소나의 병렬 시뮬레이션이 시작된다.
2. 전체 진행률(완료/진행중/대기)과 예상 소요 시간이 표시된다.
3. 개별 페르소나의 현재 상태를 볼 수 있다: 어떤 페이지, 무엇을 하는 중, 현재 감정.
4. 특정 페르소나 클릭 시 시뮬레이션 라이브 뷰(Story 5)로 진입한다.
5. 기술 문제(페이지 로딩 실패, 로그인 만료 등) 발생 시 알림이 오고 해당 페르소나만 일시정지된다.
6. 시뮬레이션 완료 시 이메일/인앱 알림을 받는다.

## Story 5: 시뮬레이션 라이브 뷰 ★

> **목업 Screen 1 — 핵심 WOW 포인트**
> AI가 실제 제품을 사용하고 있다는 "증거"를 보여주는 화면.

**Title:** 시뮬레이션 라이브 뷰 (Action Trace + 제품 화면 + 페르소나 리얼리티 엔진 State)

**Description:** As a 제품 책임자, I want to 특정 페르소나가 내 제품을 사용하는 과정을 Action Trace, 실제 제품 화면, 페르소나 리얼리티 엔진 상태와 함께 보고 싶다, so that "AI가 진짜 제품을 쓰고 있다"는 것과 "왜 그렇게 행동하는지"를 동시에 확인할 수 있다.

**Acceptance Criteria:**
1. **3-column 레이아웃**으로 구성된다:
   - 좌측: Action Trace (시간순 행동 로그)
   - 중앙: 해당 시점의 실제 제품 스크린샷
   - 우측: 페르소나 리얼리티 엔진 State
2. Action Trace 각 항목에는 타임스탬프, 행동 유형(click/scroll/type/wait/exit), 타겟 요소, 행동 이유가 표시된다.
3. 각 행동 옆에 해당 시점의 감정 상태가 이모지로 표시된다.
4. 중앙 제품 스크린샷에서 문제 지점(이해 불가, 미지 용어 등)이 어노테이션으로 하이라이트된다.
5. 우측 페르소나 리얼리티 엔진 패널에 현재 선택된 행동 시점의 상태가 표시된다:
   - **L2 인지**: 정보 향기(Information Scent), 인지 부하(%), 멘탈 모델 Gap, TAM 유용성
   - **L3 감정**: 현재 감정 라벨, PAD 3축 값(Pleasure/Arousal/Dominance), 변동성(σ)
   - **L4 의사결정**: BDI-E(Belief/Desire/Intent), Fogg Check(Motivation × Ability × Prompt) 결과
   - **L5 메모리**: Observation + Reflection 엔트리 (importance, recency 스코어 포함)
6. Action Trace에서 특정 행동을 클릭하면 중앙 스크린샷과 우측 페르소나 리얼리티 엔진 상태가 해당 시점으로 전환된다.

## Story 6: 구조화된 설문 결과 확인 ★

> **목업 Screen 5 — 30명을 정량적으로 비교하는 화면**
> UXAgent의 Survey Responses를 발전시킨 것. 고객 커스텀 질문 + Memory Stream 기반 답변.

**Title:** 구조화된 설문 결과 확인

**Description:** As a 제품 책임자, I want to 30명 페르소나의 표준 설문(SUS) + 커스텀 질문 응답을 세그먼트별로 비교하고 싶다, so that "어떤 유저 그룹이 어떤 점에서 불만족하는가"를 정량적으로 파악할 수 있다.

**Acceptance Criteria:**
1. 전체 SUS 점수가 세그먼트별로 집계되어 표시된다. (예: 코어 74/100, 엣지 48/100, 와일드카드 28/100)
2. SUS 10문항 각각에 대해 세그먼트별 평균 점수 바 차트가 표시된다.
3. 특정 문항을 클릭하면 개별 페르소나의 점수 + 이유(Memory Stream 기반 서술)가 펼쳐진다.
4. 이유 서술에 페르소나 리얼리티 엔진 수치가 근거로 인용된다. (예: "cognitive_load: 85%", "ability: 0.2")
5. 고객이 추가한 커스텀 질문(NPS, 가격 만족도 등)도 같은 형식으로 표시된다.
6. 설문 결과를 CSV/PDF로 내보낼 수 있다.

## Story 7: 리포트 요약

> **목업 Screen 3 — 증거를 본 후에 보는 요약**

**Title:** 시뮬레이션 결과 리포트 요약

**Description:** As a 제품 책임자, I want to 시뮬레이션 결과를 한눈에 파악하고 싶다, so that "이 제품을 지금 출시해도 되는가"를 판단할 근거를 얻을 수 있다.

**Acceptance Criteria:**
1. 리포트 최상단에 출시 준비도 판정(Go / 조건부 / No-Go)과 근거 요약이 표시된다.
2. 핵심 메트릭 4개가 표시된다: 핵심 플로우 완료율, 평균 만족도, 페르소나 수, 치명적 이탈 지점 수.
3. 핵심 이탈 지점 Top 3가 이탈률과 함께 표시되고, 각 지점에 이유(페르소나 리얼리티 엔진 근거 포함)와 개선 제안이 제공된다.
4. 세그먼트별 × 플로우별 만족도 히트맵 매트릭스가 제공된다.
5. 리포트를 PDF로 내보내거나 링크로 팀원에게 공유할 수 있다.

## Story 8: 감정 궤적 분석

> **목업 Screen 4 — 세그먼트 간 차이를 시각적으로 보여주는 화면**

**Title:** 세그먼트별 감정 궤적 타임라인

**Description:** As a 제품 책임자, I want to 세그먼트별 감정 변화를 플로우 단계별로 비교하고 싶다, so that "어느 지점에서 어떤 유저 그룹이 이탈하고 왜 그런지"를 시각적으로 파악할 수 있다.

**Acceptance Criteria:**
1. X축은 플로우 단계, Y축은 감정 상태(PAD Pleasure 축, -1.0~+1.0)인 라인 차트가 제공된다.
2. 세그먼트별(코어/엣지/와일드카드) 라인이 겹쳐서 "어디서 갈리는지" 한눈에 보인다.
3. 세그먼트 간 격차가 최대인 지점이 어노테이션으로 하이라이트된다.
4. 이탈이 발생한 지점에 이탈 마커가 표시된다.
5. 차트 하단에 핵심 인사이트(격차 원인 + 이론적 근거 + 개선 제안)가 제공된다.

## Story 9: 페르소나 후속 인터뷰 ★

> **목업 Screen 2 — "왜?"를 직접 물어보는 화면**
> UXAgent의 Interview 기능을 발전시킨 것. Memory Stream 기반 체험 기억 응답.

**Title:** 체험 기반 페르소나 인터뷰

**Description:** As a 제품 책임자, I want to 제품을 써본 AI 페르소나에게 직접 후속 질문을 하고 싶다, so that 리포트만으로는 알 수 없는 "왜?"를 파고들 수 있다.

**Acceptance Criteria:**
1. 좌측에 페르소나 프로필이 표시된다: 이름, 나이, 직업, 세그먼트, JTBD, Big Five 바, 세션 결과 요약.
2. 우측에 채팅 인터페이스가 열린다.
3. 페르소나는 Memory Stream(L5)의 observation/reflection을 기반으로 답변한다.
4. 각 답변 하단에 참조된 memory 엔트리(importance 스코어 포함)가 표시된다.
5. 페르소나는 자기 성격(Big Five)에 맞는 말투와 관점으로 일관되게 응답한다. (예: 낮은 개방성 → 새로운 것에 소극적 어조)
6. 체험하지 않은 내용에 대해서는 "그건 안 해봐서 모르겠어요"라고 답한다.
7. 인터뷰 내용은 자동 저장되고 리포트에 첨부할 수 있다.

## Story 10: SNS 커뮤니티 시뮬레이션 ★

> **목업 Screen 6 — "이 제품이 시장에서 퍼질까?"에 답하는 화면**
> 개별 시뮬레이션 후, 페르소나들이 SNS에서 제품에 대해 포스팅하고 서로 토론하는 시뮬레이션.

**Title:** SNS 커뮤니티 반응 시뮬레이션

**Description:** As a 제품 책임자, I want to AI 페르소나들이 제품 사용 후 SNS에서 서로 의견을 나누고 토론하는 모습을 보고 싶다, so that "이 제품이 시장에서 입소문을 탈 수 있는가"와 "어떤 담론이 형성되는가"를 사전에 예측할 수 있다.

**Acceptance Criteria:**
1. 개별 시뮬레이션(Story 5) 완료 후, 페르소나들이 SNS 피드에 제품 후기를 포스팅한다.
2. 각 포스트에는 페르소나 프로필(이름, 세그먼트 뱃지), 체험 기반 후기 텍스트, 좋아요/댓글/공유 수가 표시된다.
3. 다른 페르소나들이 포스트에 댓글로 반응한다: 공감, 반박, 팁 공유, 추가 질문 등.
4. 댓글은 각 페르소나의 성격(Big Five)과 체험 기억(Memory Stream)에 기반하여 자연스럽게 생성된다.
   - 예: 신중형이 "/" 사용법 팁 공유 → 불안형이 "진짜요??" → 비판형이 "그걸 알아야 쓸 수 있는 게 문제"
5. 우측 사이드바에 커뮤니티 인사이트가 표시된다:
   - 전체 감성 비율 (긍정/중립/부정)
   - NPS 점수
   - 입소문 예측 (강함/보통/약함)
   - 가장 많이 언급된 키워드 랭킹
   - 핵심 발견 요약 (예: "사용법 전파가 제품이 아닌 커뮤니티에 의존하는 구조")
6. 포스트는 반응 수(좋아요+댓글) 기준으로 정렬되어 가장 영향력 있는 의견이 상단에 위치한다.

## Story 11: 개선 후 재검증

**Title:** 제품 개선 후 재검증 실행

**Description:** As a 제품 책임자, I want to 문제를 수정한 뒤 동일 페르소나로 다시 테스트하고 싶다, so that 개선이 실제로 효과가 있었는지 before/after로 비교할 수 있다.

**Acceptance Criteria:**
1. "재검증 실행" 시 이전과 동일한 페르소나 구성이 기본값으로 설정된다.
2. URL이나 테스트 계정 변경 시 업데이트할 수 있다.
3. "이전 vs 이번" 비교 리포트가 자동 생성된다.
4. 개선된 지점(이탈률 감소, 감정 개선)과 퇴행된 지점이 하이라이트된다.
5. 동일 페르소나의 이전/이후 행동을 나란히 비교할 수 있다.
6. 검증 히스토리가 타임라인으로 쌓인다.
7. SNS 커뮤니티 시뮬레이션도 재실행되어, 개선 전/후 담론 변화를 비교할 수 있다.
