## 왜 이 문서를 참고하는가

이 사업계획서는 **TIPS 프로그램에 실제 합격한 최종본**이다. 우리 Personica의 예비창업패키지 사업계획서 작성 시, 다음을 참고한다:
- **구조와 흐름**: 섹션 배치, 스토리라인 전개 방식
- **시각자료 활용**: 차트, 다이어그램, 수식, 스크린샷 등의 배치와 밀도
- **기술 설명 수준**: 알고리즘/아키텍처를 비전문가에게 전달하는 방식
- **정량 지표 제시**: 매출 추정, KPI, 목표 산정 근거의 구체성
- **운영사 연계**: 대기업 운영사와의 협업 계획 서술 방식

## II. 섹션별 상세 분석

### 1. 시장현황 및 문제점 (p3~p9)

#### 구조 흐름
```
배경 (거시 문제) → 시장현황 (기존 솔루션의 한계) → 자사 서비스 소개 → 자사 서비스의 문제점 (해결해야 할 기술 과제)
```

#### 배경 (p3)
- **거시적 사회 문제로 시작**: 서울시 전체 민원의 40%가 주차 관련 (1,433,732건)
- **사회적 비용 수치화**: 연간 9조원 이상의 사회적 비용
- **악순환 구조 다이어그램**: 물리적 공간 부족 → 수요/공급 비대칭 → 임차료 상승 → 불법주차 → 다시 공간 부족
- **결론**: "주차문제는 해결이 불가한 고질적 사회적 경제적 문제"

**참고 포인트**: 구체적 수치 + 악순환 다이어그램으로 문제의 심각성과 구조적 특성을 동시에 전달

#### 시장현황 (p4)
- **기존 솔루션 3개 비교** (카카오 T주차, 맵주차, 모두의주차장)
- 각각의 **한계점을 구체적으로 나열**
- **핵심 인사이트**: "사용자에게 '주차'라는 행위 자체가 불편을 초래함"
- **해결 방향 제시**: "주차대행 서비스를 통해 근본적 문제 해결 가능"

**참고 포인트**: 경쟁 서비스를 깎아내리지 않고, "근본적 문제를 못 풀고 있다"는 프레이밍으로 자사의 차별점을 부각

#### 서비스 소개 (p5~p6)
- **핵심 가치 3가지**: 수요/공급 비대칭 해소, 저렴한 비용, 근본적 주차 고민 해소
- **실제 앱 스크린샷 5장**: 주차호출 → 차량인수 → 주차완료 → 출차호출 → 차량인계
- **트랙션 데이터**: AOS/iOS 다운로드 51,321건, 누적 주차대행 1,088건
- **구체적 가격**: 강남 기준 3시간 9,900원

**참고 포인트**: 실제 동작하는 서비스의 스크린샷 + 초기 트랙션 수치를 함께 제시하여 실행력 입증

#### 서비스 문제점 (p7~p9) — 기술 과제 도출
3가지 문제점을 명확히 정의하고 각각이 왜 기술 개발이 필요한지 연결:

| 문제 | 현황 | 필요한 기술 |
|------|------|----------|
| (1) 실시간 매칭의 한계 | 매칭 성공률 65.8%, 호출취소 23.2% | 스케줄링 기반 수요 예측 매칭 알고리즘 |
| (2) 지역 확장 시 비용 과다 | 1개 서비스존 테스트에 920만원 소요 | 교통 예측 데이터 기반 시뮬레이터 |
| (3) 제휴사 연동 지연 | API 비표준, 매번 새로 개발 | SDK 개발 |

**참고 포인트**: 문제점에 **정량 데이터**(매칭 성공률 파이차트, 테스트 비용 상세 내역)를 넣어 기술 개발의 당위성을 객관적으로 입증. 이 섹션이 Solution 섹션으로의 **자연스러운 브릿지** 역할.

### 2. 해결방안 및 세부내용 (p10~p17)

#### 기술개발 요약 (p10)
- **PIS (Parking service Integration System)**: 3개 기술 모듈로 구성
  1. 매칭 알고리즘 — 예약 Data 스케줄링 + 운영 Data 기반 수요 예측
  2. 시뮬레이터 — 이동/교통량 예측 데이터 API 기반
  3. SDK — 다른 서비스 플랫폼과 연동 가능

- 각 기술 → 비즈니스 효과 매핑:
  - 매칭 알고리즘 → UX 개선
  - 시뮬레이터 → 운영비 지출 절감
  - SDK → 시장 확장

**참고 포인트**: 기술 3개를 하나의 통합 시스템(PIS)으로 묶어 명명. 각 기술이 어떤 비즈니스 문제를 푸는지 1:1 매핑.

#### 기술개발 상세 (p11~p13)

**(1) 매칭 알고리즘 (p11)**
- 3단계 파이프라인: 교통량 예측 데이터 → 운행 데이터 추출/연산 → 링커 운영 스케줄링
- **수학 수식 포함**: 주차 완료 소요시간 연산 알고리즘 (A_L, A_P, T_L, T_P, S_L, S_P 등)
- API 제공사 명시: PlayMap, Tmap, Naver Maps
- 기술 스택: MySQL, CodeIgniter, Firebase

**참고 포인트**: 수학 수식을 넣어 **기술의 깊이를 시각적으로 보여줌**. 비전문가도 "이건 진지한 알고리즘이구나"라고 느끼게 함.

**(2) 시뮬레이터 (p12)**
- 4단계 프로세스: 좌표 설정 → 거리/시간 확인 → 운영 시뮬레이션 반복 → 결과 확인
- **반복 계산 규모**: 1,075,200회 반복 테스트
- 셀프 설정 기능: 주차장 기준 30m 단위 섹터, 최대 600m/1200m

**(3) SDK (p13)**
- 3단계 진화: 서버 연동 SDK → User UI SDK → Custom UI SDK
- SDK 없는 경우 vs 있는 경우 비교 다이어그램

#### 기술개발 목표 (p14)
| 주요 성능지표 | 최종 목표 | 가중치 | 측정방법 |
|-------------|---------|--------|---------|
| 매칭 알고리즘 | 1종 | 20% | SW 시험인증연구소 |
| 매칭 성공률 | 80% | 20% | 자체평가 |
| 시뮬레이터 | 1종 | 20% | SW 시험인증연구소 |
| 시뮬레이터 기반 지역 확장 | 3개구 | 10% | 자체평가 |
| SDK | Android/iOS 2종 | 10% | SW 시험인증연구소 |
| 부가서비스 제휴 | 2개사 연동 | 10% | MOU/NDA 서류 |
| 지적재산권 | 2건 | 10% | 특허청 출원수 |

**참고 포인트**: 목표를 정량화하고, 각각의 **측정 방법과 가중치**를 명시. 제3자 인증(SW 시험인증연구소)을 활용하여 객관성 확보.

#### 기술개발 현황 (p15~p16)
- **개발 타임라인**: 2018.12 ~ 2020.03 (약 15개월)의 버전별 개발 이력
- **실제 플랫폼 스크린샷**: 유저앱, 링커앱, 관리자 플랫폼
- **인프라 상세**: 서버 스펙(Ubuntu, 8core, 64GB RAM), 개발 환경(PHP, Java, PhpStorm)
- **운영사 협업 현황**: 현대엠앤소프트 PoC 수행, PlayMap API 연동

#### 업무분장 및 추진일정 (p17)
- **간트 차트**: 1차년도(6개 과제) + 2차년도(5개 과제)
- 수행기관: (주)마지막삼십분 100%

### 3. 사업화 전략 (p18~p35)

#### 목표시장 및 경쟁사 (p18~p21)

**(1) 글로벌 시장 (p18)**
- 스마트 주차 시장 CAGR 14% (2018~2023)
- 시장 성장 그래프 + 산업 생태계 다이어그램 (Frost & Sullivan 출처)

**(2) 유사 서비스 기업 (p19~p20)**
- **LUXE** (미국): 2013년 설립, 누적 투자 79M$, 2017년 Volvo 인수 ($140M)
- **e-Dai Bo** (중국): 2014년 설립, 누적 투자 35M$, 미쉐린 투자 유치

**참고 포인트**: 해외 유사 서비스의 **EXIT 사례**(Volvo 인수)와 투자 규모를 제시하여 시장 검증. "이 모델이 해외에서는 이미 증명되었다"는 논리.

**(3) 국내 시장 (p21)**
- 2018년 추산 1.4조 → 2020년 추산 5조
- 뉴스 기사 인용으로 시장 성장 뒷받침

#### 사업화 계획 (p22~p27)

**목표시장 세분화 (p22)**
- B2C: 개인고객 중심, 서울 경인 지역, 빠른 시장 확장
- B2B: 기업고객 중심, 대형 쇼핑몰/오피스, 매출 증대
- B2B2C: 주차장 운영기업, 종합 주차 서비스 플랫폼

**세분시장 진입 전략 (p23)**
- B2C → B2B → B2B2C 순차 진입
- 3개년 로드맵 (2020~2022)

**시장별 상세 계획 (p24~p26)**
- B2C: 서비스존 20개, 회원 15만명, 일 주차 750건, 연매출 14억
- B2B: 제휴 8개사, 일 360건, 전문 드라이버 20명, 연매출 5억
- B2B2C: 부가서비스 2개 제휴, B2B+B2C 통합 플랫폼

**마케팅 계획 (p27)**
- 분기별 목표 (2020 3Q ~ 2021 2Q)
- B2C/B2B 각각의 주요추진 목표 + 달성 계획

**참고 포인트**: 시장을 3개로 세분화하고, 각각에 대해 **별도의 로드맵과 KPI**를 제시. 단순 "시장이 크다"가 아니라 "어떤 순서로, 어떤 목표로 진입할 것인가"를 구체적으로 보여줌.

#### 운영사 지원계획 (p28~p32)
- **현대자동차** 운영사와의 3단계 협업 계획
  1. 기술 개발 및 사업화 지원 (2020하반기~2021상반기): 기아 Developers API 공유
  2. 사업화 확대 지원 (2021상반기~하반기): 현대 기아 사옥 주차대행 파일럿
  3. 기술 고도화 및 서비스 확장 (2021하반기~2022상반기): 현대엠앤소프트 협업
- **구체적 연동 시나리오**: 기아 전장 내비게이션에서 잇차 서비스 호출 플로우
- **데이터 공유 계획**: Mobility Data / Human Data / Market Data

**참고 포인트**: 운영사와의 협업을 **구체적 기술 연동 시나리오**와 **데이터 공유 구조**까지 내려가서 설명. 단순 MOU가 아닌 실제 기술적 시너지를 보여줌.

#### 고용창출 및 기대효과 (p33)
- 5년간 18명 신규 고용 계획 (9명 → 25명)
- 2024년 기준 인력 구성: App 6, 서버 2, 디자인 2, WEB 2, 기획/운영 11, 경영지원 2 = 25명
- 사회적 기대효과 나열

#### 사업화 목표 (p34)
- 매출 추정: 2022년 68.8억 → 2027년 380.5억
- 연구개발결과물 제품 점유비율 100%

#### 사업화 목표 산정 근거 (p35)
- **매출 산출 수식을 항목별로 분해**:
  - B2C: 1차량 평균 매출 × 일 주차 수행 건수 × 월간 영업일 × 12개월
  - B2B: 동일 구조, 제휴업체 기준
  - B2B2C: 차량 평균 매출 × 연간 부가서비스 수행건수
- 각 변수별 연도별 성장 가정 명시

**참고 포인트**: 매출 추정의 **모든 변수와 가정을 투명하게 공개**. 심사위원이 "이 숫자가 어디서 나왔는지" 추적할 수 있게 함.

## III. 사업계획서 작성 시 참고할 패턴

### 1. 스토리 구조
```
사회적 문제 (공감) → 기존 솔루션의 한계 (긴장) → 우리 서비스 (반전)
→ 우리도 아직 못 푼 문제 (솔직함) → 이번 기술 개발로 해결 (TIPS 당위성)
→ 사업화 계획 (비전) → 팀 (실행력)
```

### 2. 시각자료 밀도
- **전체 40페이지 중 텍스트만 있는 페이지: 거의 0개**
- 모든 페이지에 차트, 다이어그램, 스크린샷, 표, 수식, 사진 중 최소 1개 포함
- 특히 기술 섹션에서 **수학 수식**을 넣어 기술 깊이 시각화

### 3. 정량 지표의 일관성
- 문제 → 수치로 입증 (민원 건수, 사회적 비용)
- 현황 → 수치로 입증 (다운로드 수, 대행 건수, 매칭 성공률)
- 목표 → 수치로 설정 (가중치 + 측정방법 명시)
- 매출 → 수식으로 도출 (변수별 가정 투명 공개)

### 4. 운영사 활용
- 단순 "지원받겠다"가 아닌, **구체적 기술 연동 시나리오** 제시
- API 연동, 데이터 공유, 파일럿 테스트 등 **실행 가능한 계획**
- 운영사의 브랜드(현대/기아)를 서비스 신뢰성에 활용

### 5. 경쟁사 분석 방식
- 국내 경쟁사: "근본적 문제를 못 풀고 있다" 프레이밍 (깎아내리지 않음)
- 해외 유사 서비스: EXIT 사례와 투자 규모로 **시장 모델 검증**
- "우리만 할 수 있다"가 아닌 "이 모델은 해외에서 검증됨, 국내에는 우리가 유일" 논리

### 6. 팀 소개 구성
- 대표 + 고문단(기술/경영/운영) + 실행팀
- 고문단으로 기술/사업 신뢰성 확보
- 연혁으로 실행력(투자/사업선정/제휴) 입증
