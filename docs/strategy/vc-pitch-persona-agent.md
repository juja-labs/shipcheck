# ShipCheck 페르소나 에이전트 — VC 피칭 스크립트

## 한 줄 요약
"AI 페르소나가 실제로 제품을 써보고, 사람처럼 감정을 느끼면서 리뷰를 남깁니다."

---

## 1. 문제 (30초)

제품을 만들었어요. 출시 전에 "사람들이 이걸 좋아할까?" 확인하고 싶죠.

방법은 세 가지입니다:
- **사용자 테스트** — 10명 모집에 2주, 비용 수백만 원
- **설문조사** — "써보지도 않고" 의견을 물어보는 거라 부정확
- **그냥 출시** — 실패하면 몇 달의 개발 시간이 날아감

ShipCheck은 **네 번째 방법**입니다.

---

## 2. 솔루션 (1분)

AI 페르소나 30명이 여러분의 제품을 **진짜로 사용**합니다.

> "52세 초등학교 교사 박서연 씨가 여러분의 폼 빌더를 써봤습니다.
> 영어로 가득한 화면에 당황했고, 템플릿을 찾아 헤매다가,
> 결국 '그냥 구글 폼으로 돌아가겠다'고 포기했습니다."

이건 상상이 아닙니다. AI가 실제 브라우저를 열고, 클릭하고, 타이핑하면서 나온 결과입니다.

---

## 3. "GPT한테 리뷰 써달라고 하면 되는 거 아니에요?" (1분)

가장 많이 받는 질문입니다. 답은 **"그러면 모든 리뷰가 칭찬이 됩니다."**

ChatGPT에게 "52세 교사로서 이 제품을 평가해줘"라고 하면:
- 유용성 점수: **0.7/1.0** (관대함)
- 이탈 위협: **0번**
- "개선하면 좋겠지만 전반적으로 좋은 제품입니다" (sycophancy)

ShipCheck의 페르소나 에이전트는:
- 유용성 점수: **0.3/1.0** (현실적)
- 이탈 위협: **4번** ("한 번만 더 막히면 구글 폼으로 갈 거야")
- 매몰 비용 오류까지 자연 발생 ("이미 너무 와서 나가기도 그렇고...")

이 차이가 나는 이유가 저희 핵심 기술입니다.

---

## 4. 어떻게 다른가 — "감정 엔진" (1분 30초)

일반 AI는 이렇게 동작합니다:
```
"이 제품 써봐" → AI가 알아서 판단 → "좋아요!" (sycophancy)
```

ShipCheck은 이렇게 동작합니다:
```
페르소나가 클릭 → 실패함
    → 감정 엔진이 수학적으로 계산: "짜증 수치 -0.15"
    → 이 짜증이 누적되면 AI가 강제로 부정적 상태에 놓임
    → "이건 아니다, 포기하겠다"
```

핵심은 **"감정을 AI가 결정하는 게 아니라, 수학이 결정한다"**는 겁니다.

버튼을 눌렀는데 안 되면 → 짜증 수치가 올라가는 건 **공식**입니다. AI의 기분이 아니에요. 그리고 이 짜증이 쌓이면 포기하는 것도 **기준선을 넘었을 때 자동 판정**입니다. AI가 "계속 써볼까 말까" 고민하는 게 아닙니다.

이걸 저희는 **"수학이 틀을 잡고, AI가 그 안에서 행동한다"**라고 설명합니다.

---

## 5. 검증 결과 (30초)

두 가지를 검증했습니다.

**검증 1: 감정 엔진이 정말 차이를 만드는가? (Ablation Test)**

| | 감정 엔진 있음 | 감정 엔진 없음 (일반 AI) |
|---|---|---|
| 제품 평가 점수 | 0.3~0.5 | 0.6~0.8 |
| "포기하겠다" 언급 | 4번 | 0번 |
| 매몰 비용 심리 | 자연 발생 | 없음 |

→ 감정 엔진이 없으면 AI는 제품에 관대합니다. 있으면 현실적입니다.

**검증 2: 페르소나마다 다르게 행동하는가?**

같은 제품(Tally.so)을 두 페르소나에게 줬습니다:

| | 박서연 (52세 교사) | 김도현 (34세 PM) |
|---|---|---|
| 전략 | 템플릿 찾아서 안전하게 | 에디터 직행, 고급 기능 탐색 |
| 비교 대상 | "구글 폼이었으면..." | "Typeform보다 못한데?" |
| 이탈 이유 | "삭제 버튼 누르면 다 사라질까봐 무섭다" | "조건부 로직이 드롭다운 3개가 전부라고?" |
| 이탈 스텝 | 7스텝 | 12스텝 |

→ 같은 제품인데 **완전히 다른 경험, 다른 이유로 이탈**합니다.

## 6. 시장 (30초)

- 사용자 테스트 시장: $2B+ (UserTesting.com 등)
- 타겟: MVP/프로토타입을 만든 스타트업, 출시 전 검증이 필요한 팀
- 기존 대안은 비용(수백만 원) + 시간(2주+)이 진입장벽
- ShipCheck은 **URL 하나 넣으면 2시간 안에 30명의 리뷰**

---

## 7. 비즈니스 모델 (20초)

- 리포트당 과금 또는 월 구독
- 페르소나 30명 × 제품 1개 = 1회 리포트
- 고객: Pre-launch 스타트업, 프로덕트 팀, UX 리서치 팀

---

## 8. 현재 단계 (20초)

- Discovery 단계: 핵심 가설("시뮬레이션이 실제 리뷰와 유사한가") 검증 중
- 감정 엔진 + 페르소나 분화: **검증 완료**
- 실제 리뷰(G2) 유사도: **검증 진행 중** (다음 주 결과)
- 예비창업패키지 신청 준비 중

## Q&A 대비

**Q: 경쟁사는?**
A: Helio, Synthetic Users, UserTesting — 전부 "설문 기반"이거나 "실제 사람". AI가 **직접 제품을 사용하면서** 감정까지 시뮬레이션하는 건 저희뿐입니다.

**Q: 기술적 해자는?**
A: 코드 자체보다 **방법론**입니다. "어떤 심리 모델을 조합해야 현실적 결과가 나오는지"를 시행착오로 찾았고, ablation으로 증명했습니다. 그리고 고객 데이터가 쌓이면 캘리브레이션이 정밀해지는 **데이터 해자**가 형성됩니다.

**Q: 정확도는?**
A: G2 실제 리뷰와 비교 검증 중입니다. 현재까지 "영어 불편", "조건부 로직 부족", "가입 없이 시작 가능" 등 G2의 주요 테마를 합성 페르소나가 독립적으로 도출했습니다.

**Q: 왜 지금?**
A: LLM의 롤플레이 능력이 2024-2025에 급격히 좋아졌고, 브라우저 자동화 도구(Playwright)가 성숙했습니다. 1년 전에는 이 품질이 불가능했습니다.
