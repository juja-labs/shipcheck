## Executive Summary

Personica를 "제품 체험 테스트 도구"에서 **"합성 페르소나 리서치 플랫폼"**으로 포지셔닝을 확장하면:

| 지표 | 제품 체험 only | 리서치 플랫폼 | 배수 |
|------|---:|---:|:---:|
| 장기 총기회 | $0.52B | $12.6B~$14.5B | **24-28x** |
| SAM | $0.15B | $2.5B~$3.5B | **17-23x** |
| SOM (3년) | $3-5M | $15-30M | **5-6x** |
| SEO 키워드 클러스터 | 1-2개 | 7개 | **4-5x** |
| 잠재 고객/년 | ~3,200 | ~13,000-17,000 (Phase 1) | **4-5x** |

> 주의: 예창패와 공식 시장규모 문서에서는 `TAM = $5.18B~$5.86B`, `SAM = $0.91B~$1.65B`, `SOM = $4.6M~$16.5M`을 source of truth로 사용한다. 이 문서의 상단 비교표는 `제품 체험 only` 대비 `리서치 플랫폼`의 전략적 확장 폭을 보기 위한 GTM용 상향 프레임이다.

**핵심 전략 변경**: Phase 1의 첫 진입점을 "제품 체험(Playwright)"에서 **"합성 서베이 + 컨셉 테스트"**로 변경. 진입 마찰 제로(텍스트 입력만), 더 넓은 직접 접근 시장, 낮은 기술 리스크.

### 1.2 합성 침투 속도 랭킹

| 순위 | 카테고리 | 침투 속도 | 근거 |
|:---:|---------|:-------:|------|
| 1 | UX 리서치 | 가장 빠름 | 48% 리서처가 합성 사용자를 주요 트렌드로 인식(Maze 2026). 상용 제품 다수 존재 |
| 2 | 컨셉 테스트 | 빠름 | "사람 반응 예측"이 본질 → 합성 적합도 높음. NielsenIQ/Ipsos AI 도입 가속 |
| 3 | 설문/서베이 | 보통 | Qualtrics "3년 내 절반 이상 AI 수행" 전망. 법적/규제 요건 서베이는 전환 느림 |
| 4 | 광고 리서치 | 보통 | Kantar: 마케터 70%+가 AI 예측 가치 인정. 감성 반응 신뢰도 증명이 관건 |
| 5 | FGI/정성 | 느림 | 인간 간 상호작용이 핵심 가치. 사전/보조 역할로 시작해 점진적 확대 |
| 6 | A/B 테스트 | 가장 느림 | 실트래픽 기반이 본질. 합성은 "사전 필터링" 보조 역할에 한정 |

### 1.3 Personica TAM/SAM/SOM

| 구분 | 규모 | 의미 |
|------|-----:|------|
| **TAM** | **$5.18B~$5.86B** | 현재도 소프트웨어 방식으로 구매되는 핵심 리서치 시장 |
| **SAM** | **$0.91B~$1.65B** | 합성 페르소나 방식이 단기적으로 직접 접근 가능한 핵심 시장 |
| **SOM (3~5년)** | **$4.6M~$16.5M** | 보수적 SAM의 0.5%~1.0% 점유 시나리오 |
| **1차 진입시장** | **$0.93M~$3.33M** | 솔로 빌더 + 리서치 인력 없는 PM 팀 기준 현재 엔트리 가격 예산 풀 |
| **전략적 총기회** | **$12.80B~$13.48B** | 서비스 예산 전환까지 포함한 장기 확장 기회 |

> GTM 문서에서는 위 수치를 공식 기준으로 사용하고, 더 넓은 카테고리 통합 수치는 `상향 시나리오`나 `장기 비전` 설명에만 제한적으로 사용한다.

## Part 4: 성장 루프 설계

### 4.1 루프별 평가

| 루프 | 실현 가능성 | 임팩트 | 총점 | 단계 |
|------|:----------:|:------:|:----:|:---:|
| **Usage Loop** (수정→재시뮬레이션) | 4 | **5** | **9.0** | Phase 1 핵심 |
| **Viral Loop** (리포트 공유→가입) | **5** | 4 | **8.5** | Phase 1 핵심 |
| UGC Loop (빌드로그/케이스) | 4 | 4 | 7.5 | Phase 1 보조 |
| Collaboration Loop (팀 확산) | 2 | 4 | 5.0 | Phase 2 |
| Referral Loop (추천 크레딧) | 3 | 2 | 4.0 | Phase 2 (선택) |

*총점 = 실현가능성 x 0.4 + 임팩트 x 0.6*

### 4.2 핵심 플라이휠

```
┌─────────────────────────────────────────────────────┐
│   시뮬레이션 품질 (5-Layer 엔진)                      │
│        ↓                                            │
│   Actionable 리포트                                  │
│        ↓                                            │
│   ┌─────────┐    ┌──────────┐    ┌──────────┐      │
│   │ Usage   │───→│ Viral    │───→│ UGC      │      │
│   │ Loop    │    │ Loop     │    │ Loop     │      │
│   │(재사용) │    │(자연공유) │    │(콘텐츠)  │      │
│   └────┬────┘    └────┬─────┘    └────┬─────┘      │
│        ▼              ▼               ▼             │
│          신규 유저 유입 → 시뮬레이션 실행               │
│                ↓                                    │
│          만족 → NPS > 40 → (루프 처음으로)            │
└─────────────────────────────────────────────────────┘
```

**Usage Loop**: 시뮬레이션 결과 → 제품 수정 → "정말 나아졌나?" → 재시뮬레이션 → 비교 리포트
**Viral Loop**: 리포트 수신 → 트윗/IH 공유 → "이거 뭐야" 반응 → 다른 메이커 가입
**두 루프의 시너지**: Usage에서 나온 before/after 결과가 Viral의 최고 소재

### 4.3 구현 우선순위

1. 리포트에 "Powered by Personica" 워터마크 + 공유 버튼 (3일)
2. 비교 모드 ("이전 vs 현재") + 개선 점수 (2주)
3. 공유 시 리포트 요약 퍼블릭 페이지 + OG 이미지 (1주)

## Part 10: 핵심 리스크

### 10.1 불변의 전제

> 모든 확장은 단 하나의 전제 위에 서 있다:
> **"5-Layer 엔진의 시뮬레이션 충실도가 실제 의사결정에 쓸 만한 수준인가?"**
> 검증되면 모든 세그먼트가 열리고, 실패하면 모든 세그먼트가 닫힌다.

### 10.2 확장 고유 리스크

| 리스크 | 설명 | 대응 |
|--------|------|------|
| **"다 할 수 있다"의 함정** | 범용 플랫폼 메시지가 "아무것도 잘 못하는 것"으로 인식 | 각 모드의 1개 킬러 use case 확보 후 확장. Phase별 순차 진입 |
| **서베이 먼저 전략의 리스크** | 합성 서베이에서 Aaru/Synthetic Users와 직접 경쟁 | 5-Layer 감정 모델이 "ChatGPT 역할극"과 체감 차이를 만들어야 함 |
| **SEO 경쟁** | 7개 클러스터 동시 공략은 1인 팀에 과부하 | Phase별 키워드 우선순위: Phase 1은 "AI concept testing" + "synthetic user testing"만 |
| **1인 팀 대역폭** | 6개 세그먼트 x 7개 모드 동시 불가 | Year 1은 인디해커+PM, 2개 모드만. 나머지는 Phase 2 이후 |
