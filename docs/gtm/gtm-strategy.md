## 1. GTM 전략 요약

### 핵심 원칙

Personica의 GTM은 **바텀업 PLG (Product-Led Growth)** 를 기본 모션으로 한다.

```
핵심 공식:
  날카로운 1개 pain point 해결
  → 실무자가 직접 발견·체험 (프리미엄)
  → 팀으로 확산 (챔피언 전파)
  → 엔터프라이즈는 pull (알아서 찾아옴)
```

이 공식은 Maze, Dovetail, Hotjar, 채널톡 등 UX/SaaS 성공 사례에서 반복 검증된 패턴이다.

### 전체 GTM 플로우

```
Phase 1a (0~6개월): 국내 메이커 → 글로벌 빌더
  "리서치 인력 없는 빌더가 직접 써보는 합성 페르소나"

Phase 1b (6~12개월): 콘텐츠 확장 + 모바일
  "검증 사례 기반 콘텐츠 → SEO/커뮤니티 장기 유입"

Phase 2 (12~18개월): UX 리서치 시장 진입
  "탐색 단계 보완재 — 대체가 아닌 확장"

Phase 3 (18개월+): 엔터프라이즈 Pull
  "찾아오는 고객 — SSO, 감사 로그, 데이터 거버넌스"
```

## 2. 타겟 세그먼트 & ICP

### Phase 1 ICP: 리서치 인력 없는 제품 빌더

| 속성 | 설명 |
|------|------|

| **핵심 pain** | "내 제품을 실제 사용자처럼 써보고 피드백 줄 사람이 없다" |
| **예산** | $49~$499/건 (PAYG) |
| **발견 채널** | Product Hunt, Show HN, Disquiet, GeekNews, GPTers |
| **구매 방식** | 셀프서브, 신용카드 즉시 결제 |

### Phase 2 ICP: UX 리서처/PM (SMB~미드마켓)

| 속성 | 설명 |
|------|------|
| **직업** | UX 리서처, PM, 디자이너 |

| **상황** | 탐색 단계 리서치 가속화 필요, 기존 도구(Maze, UserTesting)의 보완재 탐색 |
| **핵심 pain** | "리크루팅 없이 빠르게 가설을 좁히고 싶다" |

| **발견 채널** | G2, ResearchOps Slack, UXRConf, Figma 플러그인 |
| **구매 방식** | 프리미엄 → 팀 업그레이드 (챔피언 전파) |

## 3. 포지셔닝 & 메시지

### 포지셔닝 원칙

- **Phase 1**: "리서치 인력 없는 빌더를 위한 합성 페르소나 시뮬레이션"
- **Phase 2**: "탐색 단계 리서치 보완재" — 대체가 아닌 확장
- **핵심 메시지**: **"더 사람답다"**
- **금지 표현**: "5분 만에", "URL만 넣으면", 과장·허위 수치

### 세그먼트별 메시지

| 세그먼트 | 핵심 메시지 | 증거 |
|---------|-----------|------|
| 인디해커/솔로빌더 | "혼자서도 사용자 리서치를 할 수 있다. AI 페르소나가 당신의 제품을 실제로 써보고 피드백한다" | 분화 검증 데이터, G2 비교 결과 |
| 초기 스타트업 PM | "런칭 전에 다양한 사용자 반응을 미리 확인한다. 감정·인지·의사결정까지 시뮬레이션하는 구조적 접근" | 5-Layer 엔진 아키텍처, 30명 분화 검증 |
| UX 리서처 | "탐색 단계에서 가설을 빠르게 좁힌다. 5개로 압축한 뒤 진짜 유저로 검증하는 워크플로우" | 벤치마크 보고서, 감정 궤적 데이터 |

### 경쟁 차별화 메시지

| 경쟁사 유형 | 그들의 접근 | Personica 차별점 |
|-----------|-----------|----------------|
| 단순 LLM 프롬프팅 도구 (Ditto, Aaru, Simile) | 프롬프트로 페르소나 역할극 | 5-Layer 엔진: 감정(OCC+PAD+SDE) + 인지(Information Foraging, Cognitive Load) + 의사결정(BDI-E + Fogg + Satisficing) 구조적 시뮬레이션 |
| 전통 UX 리서치 도구 (Maze, UserTesting) | 실제 사용자 리크루팅 | 리크루팅 없이 즉시 시뮬레이션. 탐색 단계 보완재로 포지셔닝 |
| 설문/서베이 도구 (Qualtrics, SurveyMonkey) | 응답자 패널 + 설문지 | 합성 페르소나가 실제 제품을 사용하며 행동·감정·의사결정 로그 생성 |

### 리서처 반론 대응

| 반론 | 대응 |
|------|------|
| "진짜 유저 대체 불가" | "동의. 탐색 단계 도구이지 최종 의사결정 도구가 아닙니다" |
| "감정 뉘앙스 없다" | "5-Layer 엔진: OCC+PAD+SDE. 감정 궤적 데이터를 직접 보여드립니다" |
| "AI가 최적 경로만 따라감" | "BDI-E + satisficing gate + Fogg check → 편의적 포기가 구조적으로 발생합니다" |
| "학습 데이터 편향" | "Big Five → 수학적 파라미터 매핑(PRISM). 인구통계 스테레오타입이 아닙니다" |
| "중요 결정에 AI 못 믿음" | "가설 생성기이지 판결 기계가 아닙니다. 신뢰 수준 명시 + 검증 포인트 권고" |

엔터프라이즈는 **push하지 않는다**. Dovetail 패턴을 따른다: 알아서 찾아온다.

| 준비 사항 | 설명 |
|----------|------|
| SSO + SAML | 엔터프라이즈 보안 요건 충족 |

| 국내 검증 사례 → 글로벌 콘텐츠 전환 | 영문 사례 스터디 2편+ |

### 주요 마일스톤

```
Month 1:  국내 소프트 런칭 (Disquiet, GeekNews, GPTers)
Month 2:  글로벌 런칭 (Product Hunt, Show HN)
Month 3:  첫 유료 고객 10건+, 온보딩 최적화
Month 6:  유료 15~30건, 반복 사용 3개사+, PMF 신호 확인
Month 9:  콘텐츠 확장 (요즘IT, 브런치, 네이버 SEO)
Month 12: 유료 200~400개사, Phase 2 진입 준비
Month 18: UX 리서치 시장 진입 (G2, ResearchOps, 벤치마크 보고서)
```

## 8. KPI & 성공 지표

| 채널 | 인지 지표 | 전환 지표 |

### 퍼널 지표

```

목표 전환율:
  방문 → 가입: 15%+
  가입 → 첫 시뮬레이션: 60%+ (온보딩 품질)
  첫 시뮬레이션 → 유료: 10%+ (가치 입증)
  유료 → 반복: 30%+ (PMF 핵심)
```

## 10. 시장 규모 연계

이 GTM 전략은 아래 시장 규모 내에서 실행된다.

```
TAM  $5.18B~$5.86B (7.25조~8.20조원) — 리서치 소프트웨어
  └─ SAM  $0.91B~$1.65B (1.27조~2.31조원) — 합성 페르소나 접근 가능
       └─ SOM  $4.6M~$16.5M (64억~231억원) — 3~5년 점유
            └─ 1차 진입시장  $0.93M~$3.33M (13억~47억원) — Phase 1 비치헤드

전략적 총기회: $12.80B~$13.48B (18조~19조원)
```
