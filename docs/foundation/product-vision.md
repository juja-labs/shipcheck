# ShipCheck — Product Vision & Positioning

## 핵심 문제

제품을 만들 때 페르소나와 유저 저니맵을 기반으로 설계하지만, 실제 다양한 사용자가 쓸 때는 예상과 다르게 동작할 수 있다. 기존 검증 방법들의 한계:

| 방법 | 한계 |
|------|------|
| 설문조사 | 가설적 질문에 가설적 답변 ("쓸 것 같다" ≠ 실제로 씀) |
| 베타 테스터 모집 | 시간/비용 소모, 편향된 표본 (얼리어답터 치우침) |
| 내부 리뷰 | 만든 사람이 평가하는 구조적 맹점 |
| 기존 UX 테스트 도구 | 기능 정상 동작 여부(QA) 초점, "사용자가 가치를 느끼는가"에 답 못함 |

## 핵심 가치 제안

> "다양한 배경·감정·맥락을 가진 AI 페르소나가 제품을 직접 사용하고, 실제 사람의 리뷰와 유사한 수준의 피드백을 5분 만에 제공한다."

**ShipCheck은 QA 도구가 아니다.** QA는 앱이 "작동하는지" 확인한다. ShipCheck은 사람들이 앱을 "원하는지" 확인한다.

## QA 도구와의 관계 (Testim, Perfecto, qTest 등)

### 기술 레이어에서의 겹침
```
QA 도구:    Playwright/Selenium → UI 탐색 → 결과 기록
ShipCheck:  Playwright           → UI 탐색 → 결과 기록
```
Playwright로 앱을 탐색한다는 메커니즘 자체는 동일.

### 다른 것: 위에 올라가는 레이어

| | QA 자동화 | 탐색적 QA | **ShipCheck** |
|---|---|---|---|
| 탐색 주체 | 스크립트 | 사람 | AI 페르소나 |
| 탐색 방식 | 정해진 플로우 | 자유 탐색 | 페르소나 성격·목적 기반 자유 탐색 |
| 판단 기준 | "이 버튼이 동작하는가" | "이게 버그인가" | "나(페르소나)에게 가치 있는가" |
| 출력 | pass/fail | 버그 리포트 | 감성 피드백, 이탈 이유, 구매 의사 |

### 포지셔닝 메시지
- 경쟁 관계 아님 — 같은 인프라(Playwright)를 다른 목적으로 사용
- ShipCheck의 핵심 IP는 Playwright가 아니라 **페르소나 모델링 + 감정 시뮬레이션 + 구조적 분석**

## 우선순위 (2026-03-16 확정)

```
[핵심] 페르소나 시뮬레이션 엔진
       ├─ 페르소나 생성 (깊이 있는 모델링)
       ├─ 제품 탐색 행동 결정 (LLM + Playwright)
       └─ 감정/판단 시뮬레이션 (사용 중 실시간 반응)
              │
              ▼
[중요] 데이터 수집 & 구조화
       ├─ 행동 로그 (어디서 막혔나, 어디서 이탈했나)
       ├─ 감성 로그 (언제 만족했나, 언제 짜증났나)
       └─ Knowledge Graph에 구조화
              │
              ▼
[확장/아이디에이션] 시장 동역학 시뮬레이션
       ├─ 네트워크 기반 입소문/채택 시뮬레이션
       └─ 예비창업패키지 사업계획서 수준으로만 언급
```

## 시장 동역학 시뮬레이션에 대한 판단

### 왜 핵심에서 제외했는가
- MiroFish의 경제/주가 주제에서는 소셜 토론이 자연스러움 (의견의 영역, 정답 없음)
- 제품 사용성은 개인 체험이 메인 — "온보딩 3단계에서 막혔다"를 토론할 게 없음
- SNS 피드 추천 알고리즘은 시뮬레이션 결과를 알고리즘에 의존하게 만듦 (불필요한 변수)
- OASIS를 쓰기 위해 소셜 시뮬레이션을 끼워넣는 것은 기술 중심 사고

### 그래도 의미 있는 지점
- 채택/구매 의사결정은 사회적 영향을 받음 ("이 앱 돈 낼 만해?", "친구한테 추천할래?")
- 네트워크 기반 전파 모델(피드 알고리즘 불필요)로 구현하면 자연스러움
- MVP 이후 확장 기능으로 유지

## 경쟁 환경

| 회사 | 차별점 | ShipCheck과의 차이 |
|------|--------|-------------------|
| Synthetic Users | 인터뷰 기반 페르소나, 85% 시뮬레이션 정확도 | 실제 제품 사용 안 함 |
| Artificial Societies (YC W25, $5.35M) | 300-5,000 페르소나 소셜 시뮬레이션 | 개별 제품 체험 약함 |
| Blok ($7.5M) | AI 제품 테스팅 | 감정/반복사용 모델 없음 |
| Uxia | Think-aloud 프로토콜, 5분 결과 | 소규모, 초기 단계 |
| UXAgent (Amazon, 오픈소스) | LLM + 브라우저 자동화 사용성 테스트 | 감정 없음, 단일 세션, 쇼핑 전용 |

**ShipCheck 차별화 지점**: UXAgent의 "실제 제품 사용" + Generative Agents의 "깊은 메모리/감정" + 반복 사용 시뮬레이션. 이 조합은 아직 없음.
