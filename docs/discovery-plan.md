# Discovery Plan: Personica 전체 제품 검증

**Date**: 2026-03-16
**Product Stage**: New (프리런치, 고객 없음)
**Discovery Question**: AI 페르소나 시뮬레이션의 충실도가 실제 의사결정에 쓸 만한 수준인가?

---

## 1. Discovery Context

### 핵심 문제
제품을 만들 때 페르소나와 유저 저니맵을 기반으로 설계하지만, 실제 다양한 사용자가 쓸 때는 예상과 다르게 동작할 수 있다. 기존 검증 방법(설문, 베타 테스터, 내부 리뷰)은 비용/시간/편향 한계가 있고, 기존 UX 테스트 도구는 "기능 동작 여부"에 초점이 맞춰져 있어 "사용자가 가치를 느끼는가"에 답하지 못한다.

### Personica의 접근
성격·감정·인지·의사결정을 갖춘 AI 페르소나가 100-500명 규모로 다양한 리서치(제품 체험, 서베이, A/B 테스트, 전문가 리뷰, 광고/퍼널 분석 등)를 수행한다. 제품 체험 모드에서는 Playwright로 실제 브라우저를 조작하여 행동/감정 로그를 수집한다.

### 피봇 이후 변경 사항
- ~~"URL만 넣으면 5분 만에"~~ → 고객이 스테이징 URL + 테스트 계정 제공
- 로그인 불필요 / 필요 케이스 분리
- 계정 병렬 사용 가능 여부에 따라 분석 속도 차이
- Personica를 필요로 하는 고객이라면 테스트 환경 제공에 적극 협조할 것으로 판단

### 경쟁 환경
| 경쟁사 | 접근법 | 실제 제품 사용? |
|--------|--------|:---:|
| Blok ($7.5M) | Figma + 분석 데이터 → 인지 시뮬레이션 | No |
| Aaru ($1B 밸류에이션) | AI 설문/여론 시뮬레이션 | No |
| Synthetic Users | AI 인터뷰/설문 참가자 | No |
| Uxia (€1M) | 프로토타입/와이어프레임 합성 테스터 | No |
| UXAgent (Amazon, 오픈소스) | LLM + 브라우저 자동화 | 부분적 (쇼핑 전용, 감정 없음) |

**Personica만의 차별점**: 페르소나 리얼리티 엔진의 감정·인지·의사결정 시뮬레이션으로 페르소나 리얼리티를 확보한 유일한 플랫폼. 제품 체험 모드에서는 Playwright로 실제 브라우저를 조작하며, 서베이·전문가 리뷰·퍼널 분석 등 다양한 리서치 유형도 동일 엔진으로 수행.

## 4. Critical Assumptions

### 전체 가정 (26개) → 하나로 수렴

26개 가정을 도출한 뒤 종속 관계를 분석한 결과, **모든 것이 하나의 핵심 가설에 수렴**:

> **"AI 페르소나 시뮬레이션의 충실도가 실제 의사결정에 쓸 만한 수준인가?"**

구체적으로 두 축:
- **(1) 분화**: 페르소나 100명+가 유의미하게 다른 행동/반응을 보이는가?
- **(2) 현실성**: 그 차이 패턴이 실제 인간과 방향적으로 일치하는가?

### 종속 구조

```
★ 핵심 가설: 시뮬레이션 충실도
    │
    ├── Yes → A3(신뢰) 해결됨
    │         A5(지불 의향) 열림
    │         B4(스케일 프리미엄) 열림
    │         D1(인터뷰 가치) 열림
    │
    ├── B3(비용): 가치 증명되면 가격에 반영 가능
    ├── C1(자동 해부): nice-to-have, 고객 수동 세팅으로 대체 가능
    └── No → 피봇 또는 근본 재설계 필요
```

## 5. Validation Experiments

### 실험 1: 분화 검증 — "같은 제품, 다른 반응이 나오는가?"

**XYZ 가설**: Big Five 기반으로 생성된 50명의 AI 페르소나가 동일한 웹 제품을 사용했을 때, 세그먼트 간 행동 패턴(탐색 경로, 체류 시간, 이탈 지점, 감정 반응)이 세그먼트 내 패턴보다 통계적으로 유의미하게 다르다 (ANOVA p < 0.05).

**방법**:
1. 공개 웹 제품 2-3개 선정 (로그인 불필요, 복잡도 다양)
   - 단순: Tally 같은 폼 빌더
   - 중간: Dub.co 같은 대시보드형
   - 복잡: Notion 같은 다기능 도구 (공개 템플릿 페이지)
2. Big Five 수치를 의도적으로 분산시킨 50개 페르소나 생성
   - 10개 세그먼트 × 5명
3. 각 페르소나가 최소 Layer 1~4 엔진으로 제품을 10분간 사용
4. 수집 데이터:
   - 탐색 경로 (방문 페이지 순서)
   - 기능별 체류 시간
   - 이탈 지점 + 이탈 이유
   - 감정 궤적 (PAD 값 변화)
   - 최종 평가 (NPS 스타일 점수 + 자유 서술)

**성공 기준**:
- 세그먼트 간 탐색 경로 유사도(Jaccard) < 0.5
- 세그먼트 간 감정 궤적 분산 > 세그먼트 내 분산 (ANOVA p < 0.05)
- 같은 세그먼트 내 행동 일관성 > 0.6
- 최종 평가 점수가 세그먼트별 다른 분포

**실패 시 신호**:
- 50명이 비슷한 경로, 비슷한 감정, 비슷한 평가 → LLM 동질화 미극복
- 같은 세그먼트 내에서도 행동이 랜덤 → 페르소나 모델링이 행동에 영향 없음

**소요**: 엔진 프로토타입 2-3주, 실험 실행 1-2일, 분석 2-3일

---

### 실험 2: 현실성 검증 — "실제 인간과 방향이 맞는가?"

**XYZ 가설**: AI 페르소나의 세그먼트별 반응 패턴이 해당 세그먼트의 알려진 인간 행동 벤치마크와 방향적으로 일치한다.

**방법 (Option A — 기존 데이터 활용, 빠름)**:
1. UXAgent 논문의 SUS 실험 결과를 베이스라인으로 비교
2. NNGroup 공개 벤치마크 데이터 활용:
   - 65세+ 태스크 수행 43% 느림 → Personica 시니어 페르소나도 그런가?
   - 바운스율 44-45% → 전체 이탈률이 비슷한 범위인가?
   - 페이지 텍스트 20%만 읽음 → 스캔 패턴 재현되는가?

**방법 (Option B — 소규모 인간 비교, 정확)**:
1. 실제 인간 10-15명 모집 (UserTesting.com 등)
2. 동일 제품 사용 → 행동 로그 + 사후 인터뷰
3. AI 페르소나 중 유사 인구통계 그룹과 비교:
   - 기능별 만족도 순위: Kendall Tau correlation
   - 이탈 지점 일치율
   - 감성 분포: KL-divergence

**성공 기준**:
- Option A: NNGroup 벤치마크 3개 이상에서 방향 일치
- Option B: 기능별 만족도 순위 Kendall Tau > 0.5, 주요 이탈 지점 70%+ 일치

**소요**: Option A는 실험 1과 병행 (추가 1주), Option B는 별도 4-6주

```

                ├── LLM 동질화 → PRISM SDE/수치 매핑 강화 후 재시도
                │
                ├── 프롬프트 문제 → Chain-of-Emotion 구조 조정 후 재시도
                │
                └── 근본적 한계 → 피봇 검토
                    - Figma/스크린샷 기반 인지 시뮬레이션 (Blok 유사)
                    - 인터뷰 전용 서비스 (Synthetic Users 유사)
                    - 또는 프로젝트 자체 중단
```
