# Ⅱ. 사업개요 및 추진계획

> 전체 사업계획서 10페이지 제한을 고려한 핵심 논리 중심 압축 초안.

## [1] 사업개요 및 시장현황

페르소닉 AI는 AI 합성 페르소나의 실제 제품 체험과 사람다운 반응 생성을 기반으로 한 사용자 리서치 시뮬레이션 SaaS이다.

| 유형 | 입력 방식 | 제공 결과 |
|------|-----------|-----------|
| 합성 서베이 | 텍스트 질문 | 타깃 페르소나의 설문·인터뷰 응답 생성 |
| UI/UX 리뷰 | 화면 이미지·목업 | 감정·인지·사용성 피드백 제공 |
| 제품 체험 | 제품 URL + 테스트 계정 | 실제 사용 후 행동·감정 변화 리포트 제공 |

- 기존 리서치 방식의 구조적 한계: 설문·인터뷰 2~4주 소요, 사용성 테스트 1~2주 소요, 제품 분석 도구의 출시 후 활용 중심
- 기존 AI 합성 페르소나 서비스의 한계: 설문 응답 생성과 스크린샷 기반 평가 중심, 실제 제품 미체험 상태의 답변 생성 문제
- 페르소닉 AI의 대응 방식: 실제 제품 체험과 감정 재현 엔진의 결합, "상상으로 답하는 AI"가 아닌 "직접 체험 후 반응하는 AI" 지향

### 시장 규모

| 구분 | 규모 | 의미 |
|------|------|------|
| TAM | 약 7조~8조원 | 현재도 소프트웨어로 구매되는 핵심 리서치 도구 시장 |
| SAM | 약 1.3조~2.3조원 | 합성 페르소나 방식이 단기적으로 직접 접근 가능한 시장 |
| SOM | 약 630억~1,148억원 | 3~5년 내 확보 가능 시장 |
| 1차 진입시장 | 약 13억~47억원 | 1인 개발자·스타트업 PM팀 대상 초기 실행 시장 |

- 초기 진입시장: 1인 개발자와 리서치 인력이 없는 스타트업 PM팀 중심, 이후 국내 스타트업에서 글로벌 SaaS 시장으로 단계적 확장

### 경쟁 현황

| 기업 | 접근 방식 | 핵심 한계 |
|------|-----------|-----------|
| Blok | 이벤트 로그 + Figma 기반 제품 실험 | 기존 사용자 데이터 필요, 실제 제품 직접 체험 부재 |
| Synthetic Users | 합성 인터뷰 + 브라우저 테스트(Beta) | 실제 체험 근거가 약한 답변, 긍정 편향 문제 |
| Uxia | Figma·이미지·라이브 URL 기반 UX 테스트 | 감정·인지·의사결정의 구조적 시뮬레이션 부재 |
| Aaru | 합성 설문·예측 엔진 | 제품 직접 사용 후 사용자 경험 평가와 차이 |

- 핵심 차별점: **실제 제품 체험**과 **감정·인지·의사결정 시뮬레이션**의 결합

## [2] 기술개발 및 사업화 전략

### 핵심 기술 구조

| 기술 요소 | 역할 | 차별성 |
|-----------|------|--------|
| 실제 제품 체험형 AI 에이전트 | 웹 제품 접속, 클릭·스크롤·입력 등 행동 수행 | 설문 응답이나 스크린샷 평가가 아닌 실제 제품 체험 기반 |
| 감정 재현 엔진 | 감정·인지·의사결정의 5레이어 계산 | 그럴듯한 응답 생성이 아닌 망설임·이탈 이유 해석 |
| 멀티 LLM 통합 구조 | 역할별 모델 배분과 시뮬레이션 실행 | 품질과 비용 효율의 동시 최적화 |

- 핵심 기술 방향: 실제 제품 조작과 감정 재현 엔진의 결합
- 기대 효과: 단순 "좋다/나쁘다" 수준의 응답이 아닌 사용자 망설임과 이탈 이유 해석

### 초기 검증 현황

| 검증 항목 | 결과 |
|-----------|------|
| 분화 검증 | 30명 시뮬레이션에서 9개 세그먼트 간 유의미한 행동·감정 차이 확인 |
| 현실성 검증 | 실제 사용자 리뷰(G2) 대비 긍정 테마 90%, 불만 테마 71% 방향 일치 |
| 엔진 효과 검증 | 감정 엔진 제거 시 긍정 편향이 0.2~0.3 높아져, 엔진의 필요성 확인 |

- 현재 프로토타입 수준에서 성격 기반 행동 분화, 감정 재현 엔진, 브라우저 자동 조작, 30명 병렬 시뮬레이션 검증 완료

### 개발 및 사업화 전략

- **2026년 상반기**: 핵심 엔진 고도화, SaaS 웹 화면 개발, 리포트 UI 및 고객 입력 흐름 정리
- **2026년 하반기**: 국내 베타 오픈, 초기 고객 대상 유료 검증, 리포트 품질 개선
- **2027년**: 반복 사용 시뮬레이션, API 연동, 세그먼트 분석 강화, 글로벌 진출 준비
- **2028년**: 구독형 기능 고도화, 엔터프라이즈 기능 추가, 다국어·대규모 시뮬레이션 확장

### 공동(협업) 연구개발 계획

- 서울 AI 허브 입주기업 대상 파일럿 테스트 진행, 실제 제품 데이터를 기반으로 한 엔진 품질 고도화
- 서울대 공학연구원과의 협력을 통한 감정 재현 엔진 학술 검증 및 공동 연구 추진
- 허브의 고성능 컴퓨팅 인프라 활용, 다수의 브라우저 세션과 LLM 호출 병렬 실행, 향후 100명 이상 규모 시뮬레이션 확장

## [3] 수익모델 및 마케팅 계획

### 수익모델

| 서비스 | 가격(초기) | 대상 고객 | 비고 |
|--------|-----------:|-----------|------|
| 합성 서베이 | $39 | 1인 개발자, 초기 스타트업 | 텍스트 질문 기반 |
| UI/UX 리뷰 | $69 | PM, 디자이너, 기획자 | 화면 이미지·목업 기반 |
| 제품 체험 | $149 | 스타트업 PM, 제품팀 | URL + 테스트 계정 기반 |

- 초기 수익모델: 건별 과금(PAYG) 중심의 낮은 시장 진입 장벽
- 이후 확장 방향: 반복 사용 고객 증가에 따른 월 구독형 상품과 엔터프라이즈 연간 계약 추가
- 제품 체험 모드 운영 방식: 테스트 URL과 계정 제공을 전제로 한 고객 협조 기반 온보딩
- 건당 공헌이익률 목표: 모드별 약 68~81%, LLM 비용 하락과 엔진 최적화에 따른 점진적 개선 기대

### 마케팅 계획

- **1차 타깃**: 1인 서비스 개발자, 리서치 인력이 없는 스타트업 PM팀, 서울 AI 허브 입주기업
- **국내 채널**: 자사 랜딩페이지, 기술 블로그, 메이커로그, Disquiet, GeekNews, 서울 AI 허브 네트워크
- **글로벌 채널**: Product Hunt, 영문 기술 콘텐츠, 글로벌 SaaS 커뮤니티
- **영업 방식**: 초기 대표 주도의 고객 인터뷰와 데모 수행, 이후 반복 사용 사례 기반 추천·오가닉 유입 확대
- **확산 전략**: 허브 입주기업 파일럿 → 국내 초기 고객 사례 확보 → 글로벌 PLG 채널 확장 순의 단계적 진행
