# Personica Use Cases

**Product**: Personica — 합성 페르소나 시뮬레이션 플랫폼
**Date**: 2026-03-22

---

## 개요

Personica의 페르소나 리얼리티 엔진은 특정 use case에 묶이지 않는 범용 합성 페르소나 플랫폼이다.
이 문서는 고객이 Personica를 활용하는 핵심 시나리오를 구체적으로 정리한다.

---

## Use Case 1: 피처 의사결정 시뮬레이션

### 문제

제품 출시 전 또는 신규 기능 추가 시, "이 피처를 유저가 정말 원할까?"는 가장 비용이 큰 질문이다.

기존 방법의 한계:
| 방법 | 한계 |
|------|------|
| 사내 회의 | HiPPO(가장 높은 직급의 의견)에 의존, 유저 관점 부재 |
| 설문조사 | "둘 다 좋다" 식의 애매한 답변, 맥락 없는 선호도 |
| A/B 테스트 | 프로토타입 2개 개발 필요 → 수주 + 개발 비용 |
| FGI | 모집 1~2주 + 진행 + 분석, 소수 의견에 의존 |

### Personica가 푸는 방식

고객이 **피처 컨셉과 타겟 유저**를 입력하면, Personica가 다양한 합성 페르소나를 생성하고 각 피처 옵션에 대한 반응을 페르소나 리얼리티 엔진으로 시뮬레이션한다.

### 구체 예시: 요리 앱 — 음성 기반 vs 터치 기반

**상황**: 요리 앱에서 "요리 진행 상황 추적" 기능을 추가하려 한다. 핵심 질문: 음성 기반으로 할지, 터치 기반으로 할지?

```
고객 입력:
  - 제품: 요리 레시피 앱
  - 검토 중인 피처: 요리 진행 상황 추적
  - 옵션 A: 음성 기반 ("다음 단계", "타이머 3분")
  - 옵션 B: 터치 기반 (화면 탭으로 단계 이동)
  - 타겟 유저: 20~50대, 주 2회 이상 요리하는 사람
```

**Personica 실행 흐름**:

```
Step 1: 페르소나 생성
  타겟 기반 15~30명 생성
  - 연령, 기술 친화도, 요리 경험, 주방 환경 등으로 분화
  - Big Five → 행동 파라미터 수학적 매핑 (Layer 1)

Step 2: 시나리오별 시뮬레이션
  각 페르소나가 옵션 A(음성), 옵션 B(터치)를 각각 시뮬레이션
  - 매 상호작용마다 페르소나 리얼리티 엔진 파이프라인 실행:
    Layer 2 (인지): 손이 젖은 상태에서의 인지 부하 차이
    Layer 3 (감정): 음성 인식 실패 시 짜증 누적 vs 터치의 안정감
    Layer 4 (의사결정): "확실한 게 낫지" → satisficing 발동 여부
    Layer 5 (기억): 반복 사용 시 습관 형성 패턴

Step 3: 교차 분석 리포트
  세그먼트별 선호도, 감정 궤적, 이탈 지점, 개선 제안
```

**시뮬레이션 결과 예시**:

| 세그먼트 | 음성 선호도 | 터치 선호도 | 핵심 인사이트 |
|----------|:-----------:|:-----------:|--------------|
| 30대 기술 친화층 | 72% | 28% | 첫 경험 후 음성 선호로 전환, 학습 곡선 낮음 |
| 50대 요리 입문자 | 61% | 39% | 음성 선호하나, 인식 실패 시 이탈률 3배 |
| 40대 숙련 요리사 | 35% | 65% | 이미 손으로 하는 게 익숙, 음성은 "불필요한 복잡성" |

**→ 의사결정 인사이트**: "음성 기반을 기본으로 하되, 터치 폴백을 필수로 제공. 특히 음성 인식 실패 시 자동 터치 전환 UX가 핵심."

### 페르소나 리얼리티 엔진이 만드는 차이

단순 LLM 프롬프팅 ("이 기능 어때?" → "좋을 것 같습니다")과 달리, Personica는:

| 단순 LLM | Personica 페르소나 리얼리티 엔진 |
|----------|-------------------|
| "음성이 편할 것 같아요" | 손이 젖은 상태 → 인지 부하 계산 → 음성이 30% 낮음 |
| "좋은 기능이네요" | 음성 인식 3회 실패 → 짜증 감정 0.7 → 이탈 확률 급증 |
| 모든 페르소나 비슷한 답변 | 기술 친화도별 수용 곡선이 완전히 다름 |
| 맥락 없는 선호도 | 반복 사용 시 습관 형성 → 장기 선호도 변화 시뮬레이션 |

### 가치

- **시간**: 설문 2주, A/B 테스트 수주 → Personica 수 시간
- **비용**: 프로토타입 2개 개발 불필요
- **깊이**: 단순 선호도가 아닌 "왜 그런 선택을 하는가"까지 설명
- **세그먼트**: 타겟 유저 내 세그먼트별 반응 차이 확인

## Use Case 3: UI/UX 디자인 피드백

Figma/프로토타입 기반으로 디자인 시안에 대한 합성 페르소나의 인지·감정 반응을 시뮬레이션한다.

---

## Use Case 4: 합성 서베이/인터뷰

설문이나 인터뷰를 합성 페르소나에게 실행. 기존 설문의 "맥락 없는 답변" 문제를 페르소나 리얼리티 엔진으로 극복한다.

---

## Use Case 5: A/B 테스트 시뮬레이션

변형 간 선호도/행동 차이를 합성 페르소나로 사전 시뮬레이션. 실제 A/B 테스트 전에 유의미한 차이가 날 변형을 사전 필터링한다.

---

## Use Case 6: 광고/마케팅 리서치

광고 크리에이티브, 메시지, 카피에 대한 합성 페르소나의 감정·행동 반응을 시뮬레이션한다.

---

## Use Case 7: 퍼널 분석 시뮬레이션

전환 퍼널 각 단계별로 합성 페르소나의 이탈/전환 패턴을 시뮬레이션하고, 이탈 원인을 감정·인지 레벨에서 분석한다.

---

## Use Case 8: 전문가 리뷰 시뮬레이션

도메인 전문가 페르소나(의사, 변호사, 디자이너 등)를 생성하여 제품/서비스에 대한 전문가 수준의 평가를 시뮬레이션한다.

---

## 공통: 왜 Personica인가

모든 use case에서 Personica의 핵심 차별점은 동일하다:

1. **페르소나 리얼리티**: 페르소나 리얼리티 엔진이 감정·인지·의사결정을 구조적으로 시뮬레이션 → 단순 LLM 프롬프팅 대비 현실적 반응
2. **세그먼트 분화**: 동일 타겟 내에서도 성격·경험·맥락에 따라 유의미하게 다른 반응
3. **"왜"의 설명**: 선호도 숫자만이 아니라, 그 선택의 인지적·감정적 근거까지 추적
4. **속도와 비용**: 기존 리서치 대비 수십 배 빠르고 저렴
