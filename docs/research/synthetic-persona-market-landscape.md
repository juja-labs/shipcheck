## 1. 페르소나 생성 패러다임 분류

조사 결과, 업계의 페르소나 생성 접근법은 4가지 패러다임으로 분류된다:

| 패러다임 | 설명 | 해당 기업 |
|----------|------|-----------|
| **A. Data-grounded** | 실제 인간 데이터(인터뷰, 서베이, 행동데이터)에서 추출 | Simile, Aaru, Qualtrics Edge, Artificial Societies, Delve AI, Simulask |
| **B. Model-structured** | 심리학/행동과학 모델(OCEAN, BDI 등) + LLM | Synthetic Users, Lakmoos, Simsurveys, (Blok) |
| **C. LLM-native** | 인구통계 속성을 LLM 프롬프트에 주입 | Evidenza, Uxia, QA Panel, C5i, Helpfull, Poll the People |
| **D. Hybrid** | 위 조합 | Ditto, Ask Rally, Yabble |

## 2. 제품별 상세 분석

### 카테고리 1: 순수 합성 리서치 플랫폼 (설문/인터뷰)

#### 2.1 Simile (simile.ai)

| 항목 | 내용 |
|------|------|
| **패러다임** | D — Hybrid (Data + Foundation Model) |
| **핵심** | 실제 인터뷰 기반 "디지털 트윈" 생성, 에이전트로 리서치 |
| **타겟** | F100 엔터프라이즈 (CVS Health, 금융기관) |
| **펀딩** | $100M Series A (2026.02), Index Ventures 리드 |
| **가격** | 엔터프라이즈 ($100,000+/년 추정) |

**페르소나 생성 방법론:**
- **창업자**: Stanford "Generative Agents" 논문(2023) 저자 Joon Sung Park, Michael Bernstein, Percy Liang
- **생성 프로세스**: 10분 인터뷰로 실제 개인의 디지털 트윈 생성
  1. 인구통계/태도 설문 수집
  2. 자연어 배경 스토리 추출
  3. Memory Stream 아키텍처에 시드 메모리로 초기화
  4. 에이전트 루프: **관찰 → 메모리 검색 → 반성(reflection) → 계획 → 실행**
- **데이터 소스**: Gallup 패널 데이터 (확률 기반 전국 대표 패널) + 개별 인터뷰
- **심리 모델**: 없음 — Big Five 같은 파라메트릭 모델 대신 **서사적(narrative) 정체성** 사용
  - 예: "John Lin은 커뮤니티를 중시하는 약국 주인이고..."
- **검증**: 85%+ 정확도 (인간 응답 대비)

**ShipCheck 대비 차이점:**
- 실제 인터뷰 필수 (완전 합성 불가)
- 심리 모델 없이 서사 기반 → 구조적 행동 차이 보장 어려움
- 브라우저 기반 제품 사용 없음
- 매크로 사회 시뮬레이션 포커스 (선거, 정책, 시장)

#### 2.2 Aaru (aaru.com)

| 항목 | 내용 |
|------|------|
| **패러다임** | A — Data-grounded |
| **핵심** | 300~5000+ AI 에이전트로 인구집단 수준 예측 |
| **타겟** | 엔터프라이즈 브랜드, 컨설팅 (EY 등) |
| **펀딩** | Series A, $1B 헤드라인 밸류에이션 (2025.12), Redpoint 리드 |
| **가격** | 미공개 (ARR < $10M) |

**페르소나 생성 방법론:**
- 2.5M+ 에이전트 — 실제로는 **인구통계 세그먼테이션** (나이×성별×지역×소득 등 조합)
- "ground-truth training" = 실제 세계 결과(시장 점유율, 선거 결과 등)에서 학습
- **구체적 아키텍처 미공개** — 개별 에이전트의 심리 모델이나 생성 방법 공개하지 않음
- 본질적으로 **통계적 인구 대표체**이지, 깊은 행동 시뮬레이션은 아닌 것으로 추정
- 검증: EY 6개월 글로벌 연구를 1일만에 재현 (90%+ 상관관계)

#### 2.3 Synthetic Users (syntheticusers.com)

| 항목 | 내용 |
|------|------|
| **패러다임** | B/D — Model-structured + Hybrid |
| **핵심** | OCEAN 기반 합성 인터뷰 |
| **타겟** | 제품팀, UX 리서처, 스타트업 |
| **펀딩** | Seed (Comcast LIFT Labs, Urban Innovation Fund) |
| **가격** | 3 tier, 7일 무료 |

**페르소나 생성 방법론 (업계에서 가장 ShipCheck과 유사한 철학):**
- **심리 모델**: 명시적으로 **OCEAN (Big Five)** 사용하여 성격 특성 할당
- **Chain-of-feeling**: 응답 생성 전 감정/태도 상태를 중간 추론 단계로 생성
  - Chain-of-Thought의 감정 버전 — 감정 상태 → 응답 순서로 생성
  - ShipCheck Layer 3 (OCC + PAD + SDE)과 유사한 목적이나, 단일 추론 단계 vs 연속 상태
- **Shuffle v2**: 페르소나 유형별로 **다른 LLM에 라우팅** — 단일 모델 편향 방지
  - 라우팅 에이전트가 페르소나 특성에 맞는 최적 LLM 선택
- **핵심 불명확점**: OCEAN → 행동 파라미터 매핑의 구체적 수학적 방법론 미공개
  - "high openness"를 프롬프트에 주입하는 수준인지, PRISM처럼 수학적 매핑인지 불분명

**ShipCheck 대비 차이점:**
- 설문/인터뷰 시뮬레이션만 — 실제 제품 사용 없음
- 감정 모델이 단일 추론 단계 (연속적 PAD 상태 아님)
- 인지/의사결정 모델 레이어 없음
- 메모리 아키텍처 불분명

| 항목 | 내용 |
|------|------|
| **패러다임** | C — LLM-native + 검증 레이어 |
| **핵심** | 합성 고객으로 세그멘테이션/포지셔닝/크리에이티브 테스트 |
| **타겟** | 엔터프라이즈 마케팅/전략 (BlackRock, MS, JP Morgan, Salesforce) |
| **펀딩** | 부트스트랩 (VC 없음, 흑자) |

**페르소나 생성 방법론:**

- **데이터 기반이 아님** — 생성 후 실제 데이터와 **사후 검증**
  - Salesforce 상관관계: 0.81
  - 100+ 검증에서 88% 정확도
- 6개 리서치 모듈: Segmentation, Positioning, Creative Testing, Personas("Impersonas"), Measurement 등

#### 2.6 Ditto (askditto.io)

| 항목 | 내용 |
|------|------|
| **패러다임** | D — Hybrid (Census + Behavioral Science + Live Signals) |
| **핵심** | 인구통계 기반 30만+ 사전구축 페르소나 |
| **타겟** | PM, 마케터, 그로스팀 |
| **가격** | $50,000–$75,000/년 |

**페르소나 생성 방법론 (업계에서 가장 투명한 방법론 공개):**
- **핵심 속성**: 나이, 가구구성, 소득대, 도시/농촌, 문화/정체성
- **데이터 소스**: 공공 통계(센서스) + 승인된 고객 데이터
- **캘리브레이션**: 관찰된 실세계 결과(시장 점유율, 채택 곡선, 서베이 응답, 캠페인 실적)와 대조
- **라이브 시그널**: 매일 시장 데이터, 감성 추적, 문화 컨텍스트, 위치 데이터 업데이트
  - "작년 학습 데이터가 아닌 현재에 살아있는" 페르소나
- **행동과학**: **say-do gap** (말하는 것 vs 실제 행동 차이) 명시적 모델링
- **그룹 다이내믹스**: 합성 포커스그룹에서 페르소나 간 토론 시뮬레이션
- **검증**: 홀드아웃 테스트, 과거 데이터 백테스팅, 지역/세그먼트별 오차 추적
- **학술 자문**: Harvard, Cambridge, Stanford, Oxford, Seoul National University, Georgetown
- **설계 철학**: "Fewer, smarter dials beat a dashboard of toggles no one needs"

#### 2.20 UXAgent (Amazon / CHI 2025)

| 항목 | 내용 |
|------|------|
| **패러다임** | C/B — LLM-native + Dual Process Theory |
| **핵심** | 브라우저 자동화 + 합성 페르소나 (학술) |
| **출처** | Amazon Science, CHI 2025 논문 (arXiv:2504.09407) |

**페르소나 생성 방법론 (논문에서 상세 공개):**
- **Persona Generator 모듈:**
  - 입력: 예시 페르소나 + 타겟 인구분포 (예: "10% 저소득")
  - 시스템이 분포에서 인구통계 속성 샘플링 → LLM이 전체 페르소나 생성
  - 속성: 성별, 나이, 소득, 쇼핑 빈도, 학력, 온라인 경험, 직업
  - **심리 모델 없음** — Big Five 등 미사용, 순수 인구통계
  - 다양성 메커니즘: 이전 생성된 페르소나를 다음 생성 시 예시로 제공 (반복 방지)
  - 1,000 페르소나 ~2분 생성
- **에이전트 아키텍처: Kahneman Dual Process Theory**
  - Fast Loop (System 1, 직관) + Slow Loop (System 2, 숙고)
  - Planning Module: 페르소나의 의도와 관련 메모리 기반으로 단기 계획 생성/갱신
  - Universal Browser Connector: HTML → 간소화 DOM → 텍스트 관찰 메모리
  - 액션: click, type, hover, select, navigate
- **검증**: UX 리서처 16명이 20개 시뮬레이션 세션 평가 (정성적)
  - 에이전트 행동 vs 실제 인간 행동 정량 비교 없음 (저자가 명시적 한계로 인정)

**ShipCheck 대비 차이점:**
1. 인구통계만, 심리 모델 없음
2. 감정 상태 모델링 없음
3. 세션 간 메모리 지속 없음
4. 연구 도구, 상용 제품 아님

## 3. 학술적 기반: "Silicon Sampling" (Argyle et al., 2022)

대부분의 LLM-native 제품이 참조하는 학술 기반:
- GPT-3을 "수천 개의 사회인구학적 배경 스토리"로 조건화
- 모델이 **"알고리즘적 충실도(algorithmic fidelity)"** — 인구통계적으로 상관된 편향 재현
- "다양한 인간 하위집단의 응답 분포를 정확하게 에뮬레이트" 가능

**→ Helpfull, Poll the People, Evidenza, Uxia 등 대부분의 LLM-native 제품의 지적 기반**
