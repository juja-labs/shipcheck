## 1. 문제인식 (Problem) — 3~5페이지

> **가이드**: `bizplan-writing-guide.md` §2-1 참조
> **드라마 아크**: 감정적 훅 → 기존 시도의 실패 → 시장 분석 → 경쟁 현황 → 우리가 필요한 이유

## 2. 실현가능성 (Solution) — 3~5페이지

> **평가 기준**: 문제점 해결을 위한 개발(개선)방법, 경쟁력 확보 방안 등
> **가이드**: `bizplan-writing-guide.md` §2-2 참조
> **드라마 아크**: 솔루션 요약 → 작동 원리 → 차별성 비교 → 실험 결과 → 추진 일정

<!--

### 2-2. 개발 방법 — 5-Layer 시뮬레이션 엔진

<!--
[가이드 §2-2 ②] 작동 원리 (비전문가 버전)

필요한 콘텐츠:
- 5-Layer를 비전문가 언어로:
  Layer 1: "이 사람은 어떤 성격인가?" (페르소나 프로필)

- Playwright 기반 실제 제품 사용 설명
- 학술 근거는 간단히 언급 + 상세는 부록으로

필요한 시각자료:
- [ ] 5-Layer 파이프라인 도식 (비전문가 버전 — 사람이 제품 사용할 때 일어나는 일)
- [ ] 기술 아키텍처 다이어그램 (간결한 버전)

작성 주의:
- 심사위원이 볼 포인트: "이 사람이 정말 만들 수 있는가" + "구체적 절차"
- 복잡한 기술은 비유 활용
- OCC Appraisal, BDI-E, PAD 등 학술 용어는 부록으로
