## 1. 문제인식 (Problem)

### 1-1. 개발 동기

1) 사용자 검증의 구조적 문제

- 스타트업 실패 원인 1위: "시장이 원하지 않는 제품" (CB Insights)
- 로우코드·노코드·생성형 AI 확산으로 **제품 제작 속도는 빨라졌으나**, 사용자 검증 방식은 여전히 설문·인터뷰·베타테스터에 의존
- PM의 42%가 직접 사용자 리서치를 수행하고 있음 (Maze, 2025)
- 많은 팀이 여전히 지인 반응, 소수 인터뷰, 창업자 직감에 의존하여 의사결정

2) 대표자 문제 경험

- SSG.COM, SKT, KT에서 11년 이상 대화형 AI·생성형 AI·사용자 행동 데이터 기반 시스템 개발
- 공통 경험: 기능 구현은 가능하나, 서로 다른 사용자 맥락에서 어떤 감정과 판단이 발생하는지는 출시 전 확인이 어려움
- 특히 초기 제품·신규 기능은 데이터 부족으로 기존 분석 도구 활용 불가

3) Personica의 출발점

- **핵심 문제**: "출시 전에 다양한 사용자 반응을 사람에 가깝게 시뮬레이션할 수 있는 방법이 없다"
- **해결 방향**: 사람처럼 느끼고 판단하는 합성 페르소나로 제품 체험, 서베이, A/B 테스트, 퍼널 분석을 하나의 엔진으로 수행

### 1-2. 해결방안의 필요성

1) 기존 검증 방식의 한계

| 방식 | 장점 | 한계 |
|------|------|------|
| 설문·인터뷰 | 사용자 의견 수집 가능 | 실제 행동 확인 불가. 사후 합리화 편향 |
| 세션 분석 (Hotjar, FullStory, Amplitude) | 이탈 구간 확인 가능 | 출시 후 데이터 전제. "왜" 이탈했는지 설명 불가 |
| 인간 사용성 테스트 (UserTesting) | 실제 검증 가능 | 연간 구독·기업형 구매 구조. 시간·비용 부담 큼 |
| 범용 생성형 AI (ChatGPT, Claude) | 빠른 의견 생성 | 실제 제품 미사용. 경험 기반 정보 생성 불가 |
| 합성 리서치 (Qualtrics 합성 응답) | 초기 방향성 검증 유용 | 중요 의사결정에는 인간 검증 병행 권고 (Qualtrics 공식) |
| 제품 실험 (Blok) | 출시 전 비교 실험 가능 | Figma·기존 행동 데이터 중심. 실제 사용 과정 파악 한계 |

2) 시장 공백

- 기존 도구는 각각 **의견(설문)**, **행동 로그(분석)**, **예측(AI)**에 특화
- **"출시 전에 다양한 사용자 유형이 실제 제품을 사용하며 어떤 감정·판단을 거쳐 남거나 떠나는지"**를 구조적으로 검증할 도구는 부재
- Personica의 필요성은 이 지점에서 발생

### 1-3. 목표 시장 분석

1) 시장 규모 (TAM / SAM / SOM)

| 구분 | 정의 | 규모 |
|------|------|------|
| TAM | 온라인 설문 SW + UX 리서치 SW | $5.18B~$5.86B (7.25조~8.20조원) |
| SAM | 합성 페르소나 방식 직접 접근 가능 시장 | $0.91B~$1.65B (1.27조~2.31조원) |
| SOM | 3~5년 내 점유 목표 (SAM의 0.5~1.0%) | $4.6M~$16.5M (약 64억~231억원) |
| 1차 진입시장 | 솔로 빌더·리서치 없는 PM 기준 | $0.93M~$3.33M (약 13억~47억원) |
| 전략적 총기회 | 서비스 예산 전환 포함 장기 기회 | $12.80B~$13.48B (17.92조~18.87조원) |

- TAM 산정 기준: 리서치 업무를 소프트웨어로 제공하는 시장 (글로벌 인사이트 산업 $153B 중 현재 SW 구매 시장만 추출)
- 장기 확장: ESOMAR 기준 디지털 정량조사 중 SW 전환 가능분 $7.62B 추가

2) 시장 진입 적기 근거

- 리서처의 69%가 이미 AI를 연구 업무에 활용 중 (Maze, 2026)
- 조직의 88%가 최소 1개 기능에 AI 사용 (McKinsey, 2025)
- Qualtrics: 합성 응답에 대한 높은 수용·기대 공개

### 1-4. 경쟁 현황

1) 경쟁 구도

- 합성 리서치 시장은 이미 형성 중: Aaru(시리즈 A 대규모 유치, 2025), Blok($7.5M 투자 유치, 2025.07), Qualtrics(합성 응답자·AI 에이전트 제공)

| 경쟁군 | 대표 경쟁사 | 강점 | Personica와의 차이 |
|--------|-------------|------|------------------|
| 인간 기반 사용성 테스트 | UserTesting | 실제 사람 반응 확인 | 비용·리드타임 부담, 반복 실행 어려움 |
| 세션 분석 도구 | Hotjar, FullStory, Amplitude | 실사용 로그 수집, 이탈 구간 확인 | 출시 후 데이터 필요. 이탈 이유 설명 불가 |
| 합성 설문·리서치 | Aaru, Qualtrics | 빠른 방향성 검증, 대규모 응답 | 제품 미사용. 경험 기반 검증 한계 |
| 디자인·데이터 기반 예측 | Blok, Uxia | 출시 전 비교 실험·예측 | Figma·기존 데이터 의존. 실사용 마찰 포착 어려움 |

2) 시장 공백

- 시장은 크게 두 방향: ① 설문·인터뷰를 AI로 대체 ② 디자인·데이터 기반 행동 예측
- **실제 제품을 브라우저에서 조작 + 감정·인지·의사결정 추적**하는 방식은 상용 시장에서 비어 있음
- 학술 영역에서 UXAgent(Amazon, CHI 2025) 등 시도 있으나 특정 조건 한정
- 이 영역이 Personica의 진입 기회

## 2. 실현가능성 (Solution)

### 2-1. 최종 산출물

1) 서비스 개요

- **합성 페르소나 시뮬레이션 플랫폼**: 고객이 스테이징 URL + 테스트 계정 + 타깃 사용자 설명을 입력하면, 수백 명 규모의 합성 페르소나가 제품을 체험하고 구조화된 리포트를 제공
- 초기 주력: **제품 체험 시뮬레이션** 모드
- 확장 예정: UI/UX 디자인 피드백, 합성 서베이·인터뷰, 광고 메시지 테스트, A/B 사전 시뮬레이션

2) 고객이 얻는 핵심 가치

| 가치 | 설명 |
|------|------|
| 실제 체험 기반 피드백 | 합성 페르소나가 실제 제품을 사용한 결과를 바탕으로 피드백 생성 |
| 세그먼트별 비교 | 서로 다른 사용자 유형 간 반응 차이를 정량·정성으로 비교 |
| 실행 가능한 개선 제안 | 우선순위별 개선 포인트를 구조화하여 제공 |

3) 제품 플로우

- 고객 입력 → 제품 자동 분석(크롤링, Feature 온톨로지 생성) → 페르소나 N명 생성 → 시뮬레이션 실행 → 리포트 생성

4) 리포트 구성

- 세그먼트별 이탈 히트맵
- 감정 변화 타임라인
- 행동 로그 기반 세션 기록
- 요약 평점 및 추천 의향
- 우선순위별 개선 제안
- 선택형 페르소나 인터뷰

### 2-2. 개발 방법 — 5-Layer 시뮬레이션 엔진

1) 핵심 구조

- Personica의 핵심은 **5-Layer 시뮬레이션 엔진** — 합성 페르소나가 사람처럼 느끼고 판단하게 만드는 구조
- 제품 체험 시 브라우저를 자동 조작하는 기능은 인터페이스일 뿐, 핵심 IP는 엔진에 있음

| Layer | 역할 | 설명 |
|-------|------|------|
| L1. 페르소나 프로필 | 이 사람은 누구인가 | 나이, 직무, 성격 특성, 디지털 숙련도 → 행동 성향을 수학적으로 매핑 |
| L2. 인지 | 화면을 이해하는가 | 현재 UI가 이 사용자에게 얼마나 복잡한지, 기대와 실제 괴리를 계산 |
| L3. 감정 | 지금 어떤 기분인가 | 성공·실패·혼란 등 감정 변화를 쾌감·긴장·통제감 축으로 연속 추적 |
| L4. 의사결정 | 다음에 무엇을 하는가 | 현재 감정과 목표를 바탕으로 클릭, 탐색, 포기, 뒤로가기 등 행동 결정 |
| L5. 기억 | 이전 경험을 기억하는가 | 여러 세션에 걸쳐 익숙해지거나 실망이 누적되는 과정을 기록 |

2) 단순 프롬프트와의 차이

- 일반 AI: "사용자 역할로 답변" → 비슷한 답변 반복
- Personica: 성격 점수에서 행동 특성을 도출하고, 감정 상태가 확률적으로 변화 → **같은 오류를 겪어도 페르소나마다 다른 반응**이 구조적으로 발생

3) 제품 체험 모드 실행 루프

- 화면 읽기 → 해석(인지) → 감정 갱신 → 행동 선택 → 실행 → 상태 갱신 → 반복
- 최종 평점뿐 아니라 **"왜 그런 결과가 나왔는지"**를 시간 흐름에 따라 설명 가능

4) 개발 방향

- 브라우저 자동 조작: 이미 검증된 오픈소스 도구 활용, 안정성 확보
- 핵심 경쟁력: 감정·인지·의사결정 엔진의 정교함에 집중

### 2-3. 개발 진행 상황

1) 현재 구현 완료 모듈

| 모듈 | 상태 | 비고 |
|------|------|------|
| L1. 성격 기반 행동 파라미터 매핑 | **구현 완료** | 페르소나 리얼리티 핵심 |
| L3. 감정 시뮬레이션 엔진 | **구현 완료** | 페르소나 리얼리티 핵심 |
| L2, L4, L5 | 설계·일부 로직 확보 | 상용 수준 고도화 필요 |
| 브라우저 자동 조작 | 작동 확인 | 안정화 완료 |

2) 초기 검증 결과 — Tally.so 대상 30명 시뮬레이션

| 항목 | 결과 |
|------|------|
| 평균 평점 | 3.80 / 5.0 |
| 추천 비율 | 90% |
| 세그먼트별 평점 범위 | 3.50 ~ 4.33 (분명한 분화 확인) |
| 세그먼트 유형 | 기술숙련형, 체계형, 비평형, 입문형 등 — 같은 제품에서 다른 경로·이탈 이유 |

3) 현실성 검증 — G2 실제 리뷰 비교

| 비교 항목 | 일치율 |
|----------|--------|
| 긍정 테마 (상위 10개) | **9개 일치 (90%)** |
| 불만 테마 (상위 7개) | **5개 일치 (71%)** |
| 주요 포착 테마 | 사용 편의성, 무료 티어, 디자인 품질, 조건부 로직, 학습 곡선, 템플릿 한계 |

4) 감정 엔진 효과 검증 — 엔진 ON/OFF 비교 실험

| 항목 | 감정 엔진 OFF | 감정 엔진 ON |
|------|-------------|------------|
| 유용성·사용성 점수 | 0.2~0.3p 더 높음 (과도한 긍정 편향) | 보다 현실적인 범위 |
| "포기하겠다" 반응 | **0건** (비현실적) | 좌절·불안 누적에 따른 자연스러운 이탈 발생 |
| 의미 | AI의 긍정 편향 그대로 노출 | **구조적으로 긍정 편향 감소** |

5) 현재 상태 요약

- 핵심 엔진 가능성 검증: **완료**
- 제품 체험 모드 초기 작동: **확인**
- 리포트·대시보드 제품화: 고도화 필요
- 다중 세션 기억 기능: 다음 단계 핵심 과제
- **예창패 자금은 이미 검증된 엔진을 반복 사용 가능한 서비스로 고도화하는 데 투입**

### 2-4. 경쟁사 대비 차별성

| 비교 항목 | UserTesting | Hotjar / FullStory | Blok | Aaru / Qualtrics | **Personica** |
|-----------|-------------|---------------------|------|-------------------|---------------|
| 실제 제품 사용 | 사람 기반 가능 | 출시 후 로그만 수집 | 제한적 | 불가 | **가능** |
| 출시 전 검증 | 가능 (느리고 비쌈) | 불가 | 가능 | 가능 | **가능** |
| 감정/인지 시뮬레이션 | 인터뷰 해석 의존 | 없음 | 일부 예측 | 응답 생성 중심 | **5-Layer 구조적 시뮬레이션** |
| 초기 제품 적용 | 제한적 | 불가 | 제한적 | 가능 | **가능** |
| 세그먼트별 행동 로그 | 제한적 | 로그 있으나 해석 약함 | 예측 중심 | 응답 중심 | **행동+감정 동시 제공** |
| 확장 모드 | 테스트 중심 | 분석 중심 | 실험 중심 | 설문 중심 | **체험+서베이+A/B+퍼널+전문가 리뷰** |
