## 1. 문제 인식 (Problem)

### ㅇ 창업 아이템의 필요성

스타트업 실패 원인의 42%는 "시장이 원하지 않는 제품"이다 (CB Insights). 기술력 부족(17%), 팀 문제(23%), 자금 소진(29%)을 모두 앞선다.

```
  스타트업 실패 원인 (CB Insights)

  시장 미수요  ██████████████████████████████████████████░░░░░░░░  42% ◀ 1위
  자금 소진    █████████████████████████████░░░░░░░░░░░░░░░░░░░░  29%
  팀 문제      ██████████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░  23%
  기술 부족    █████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  17%
```

노코드·AI로 제품 개발 속도는 빨라졌는데, 사용자를 이해하는 방식은 10년째 정체되어 있다. Stripe 2024년 데이터에 따르면 인디 SaaS의 45.7%가 솔로 파운더이지만, "누가, 왜, 어디에서 이탈하는가"를 확인하는 방법은 여전히 설문과 소수 인터뷰뿐이다. 제품은 매주 바뀌고 스프린트는 기다려주지 않는데, 제대로 된 리서치는 결과가 나오기까지 8~10주가 걸린다 (Qualtrics, 2025). 리서치가 완료될 때쯤이면 이미 의사결정은 끝나 있다. 많은 팀이 결국 지인 3~5명의 반응이나 직감에 의존하게 된다.

글로벌 인사이트 산업 — 설문, 인터뷰, 분석, 컨설팅을 모두 합친 시장 — 은 $153B(약 214조원) 규모이다 (ESOMAR, 2025). 이 거대한 예산이 투입되는 기존 리서치 방식에는 왜 이런 한계가 있는 것인가.

#### 1) 설문·인터뷰·FGI — "말"을 측정하지, "행동"을 측정하지 않는다

사람은 자신의 행동을 정직하게 설명하지 못한다. 기억은 왜곡되고, 이해관계가 답변을 오염시키며, 사회적 시선이 진짜 생각을 숨긴다. 이를 학술적으로 사후 합리화 편향, 사회적 바람직성 편향이라 한다. 설문에서 "이 기능을 쓰겠다"고 답한 사람이 실제로 쓰는 비율은 이보다 훨씬 낮다. 또한 아직 출시되지 않은 신제품에 대한 설문은 근본적으로 한계가 있다. 경험하지 않은 것에 대한 의견은 추측일 수밖에 없기 때문이다.

#### 2) 세션 분석(Hotjar, 뷰저블 등) — 출시 후에만 작동하고, "왜"를 모른다

히트맵·세션 녹화 도구는 사용자가 어디를 클릭했는지는 보여주지만, 왜 클릭했는지, 왜 떠났는지는 알 수 없다. 그리고 출시 전에는 데이터 자체가 없어 사용이 불가능하다. 국내에서는 뷰저블(Beusable)이 삼성전자·LG전자·SK텔레콤 등 대기업 고객을 확보하며 이 시장을 형성해 왔으나(2022년 매출 약 30억원), 2024년 영업적자(-2.5억원)와 직원 수 정체가 보여주듯 사후 분석 방식 자체의 성장에는 한계가 있다 (혁신의숲).

#### 3) 인간 사용성 테스트(UserTesting 등) — 비싸고, 느리고, 소수만 가능하다

실제 사람이 실제 제품을 사용하는 방식은 가장 깊은 인사이트를 주지만, 비용과 시간의 벽이 높다. UserTesting 기준 연간 $46K+(약 6,400만원) 이상이며, 5~15명을 모집하고 테스트를 진행하는 데 1~2주가 걸린다. 반복할수록 비용은 쌓이고, 규모를 늘리기는 어렵다.

| 방식 | 할 수 있는 것 | 구조적 한계 | 비용·시간 |
|------|-------------|------------|-----------|
| 설문·인터뷰·FGI | 사용자 의견 수집 | 실제 행동 확인 불가, 사후 합리화 편향, 미출시 제품 한계 | 2~4주, 인당 $50~200 |
| 세션 분석 (Hotjar, 뷰저블) | 클릭·이탈 지점 확인 | "왜"를 모름, 출시 후에만 가능 | 데이터 축적 전제 |
| 인간 사용성 테스트 (UserTesting) | 실제 사용 관찰 | 연 $46K+, 5~15명, 1~2주. 반복 부담 | 초기 팀에 비현실적 |

결국 설문은 의견만 들을 수 있고, 분석 도구는 출시 후에만 작동하며, 사용성 테스트는 비싸고 느리다.

대표자의 문제 경험

대표자는 SSG.COM, SKT, KT에서 11년 8개월간 대화형 AI·생성형 AI·사용자 행동 데이터 기반 시스템을 개발하며 이 문제를 반복 경험하였다. KT에서 20명 규모 AI엔지니어링팀을 리딩하며 보험·제조·교육·금융 등 다양한 도메인에서 AI Agent를 개발했지만, 제품을 만들고 검증하려 할 때마다 기존 방식의 한계에 부딪혔다:

- 기능 구현은 가능했지만, 서로 다른 사용자가 어떻게 반응할지 확인하려 하면 기존 방식(설문, 소수 인터뷰, 분석 도구)으론 부족했다
- 설문은 의견만 들을 수 있었고, 분석 도구는 출시 전엔 쓸 수 없었고, 사용성 테스트를 하기엔 시간과 예산이 없었다
- 결국 지인 반응(3~5명), 소수 인터뷰, 직감에 의존한 의사결정

AI 페르소나라는 새로운 대안

이 구조적 한계에 대한 근본적으로 새로운 접근이 등장하고 있다. AI가 생성한 가상의 사용자(합성 페르소나)가 설문에 응답하고, 제품을 평가하고, 인터뷰에 참여하는 방식이다.

| 기존 방식의 한계 | AI 페르소나가 여는 가능성 |
|----------------|------------------------|
| 설문은 의견만 수집 (행동 불가) | 사용자 행동을 시뮬레이션 — 의견이 아닌 행동 예측 |
| 분석 도구는 출시 후에만 가능 | 출시 전에도 시뮬레이션 가능 |
| 사용성 테스트는 비싸고 느리고 소수(5~15명) | 수백 명을 수 시간 내에 저비용으로 시뮬레이션 |

이 접근은 이미 실제 서비스로 제공되고 있다. 2025~2026년 사이에만 $1B 밸류에이션(Aaru), $100M Series A(Simile), $7.5M 투자(Blok)가 이루어졌고, Qualtrics 같은 대형 설문 플랫폼도 합성 응답자를 공식 탑재하였다. EY는 합성 페르소나 1,000명과 실제 설문을 비교하여 90%+ 상관관계를 확인하였으며, 리서처의 48%가 "합성 유저"를 2026년 최대 트렌드로 선택하였다 (Maze, 2026). 주요 경쟁사의 상세 분석은 3장(성장전략)에서 서술한다.

궁극적으로 AI 페르소나 기술은 개인 응답 수집을 넘어, 수천 명의 합성 페르소나가 서로 상호작용하며 시장 수준의 반응을 예측하는 사회 시뮬레이션까지 가능하게 한다. 이는 예산이 아무리 많아도 사람으로는 불가능한, 기술이 열어주는 완전히 새로운 영역이다.

그러나 현재 AI 페르소나에는 세 가지 한계가 있다

기존 리서치의 비용·시간·규모 문제는 AI 페르소나가 해결하기 시작했다. 그러나 현재의 AI 페르소나 서비스에는 아직 세 가지 핵심 한계가 존재한다. 긍정 편향은 답의 방향이 틀리는 문제이고, 제품 미사용은 경험의 깊이가 없는 문제이며, 분화 부족은 사람마다 달라야 할 답이 획일적인 문제이다.

한계 ①: AI 특유의 긍정 편향 — "다 좋다"고 답하는 AI. AI 모델의 전체 응답 중 58%가 아부적 행동을 보이며(TechPolicy.Press, 2025), NN/g(닐슨 노먼 그룹)는 합성 유저 응답이 "너무 얕아서 실용적으로 쓸 수 없는 수준"이라고 지적하였다. 실제 사용자는 "7개 과정 중 3개만 완료했다"고 답하지만, 합성 유저는 "전부 완료했습니다"라고 답한다.

한계 ②: 제품을 실제로 사용하지 않음. 현재 주요 AI 페르소나 서비스는 설문에 답하거나 Figma 디자인을 평가하는 방식이다. 실제 제품을 써보지 않으면 "3번째 화면에서 버튼을 못 찾아서 30초간 헤맸다" 같은 경험 기반의 구체적 문제를 포착할 수 없다.

한계 ③: 분화 부족 — "다 비슷하게" 답하는 AI. 서로 다른 배경의 페르소나인데도 비슷한 패턴으로 응답한다. Qualtrics도 "범용 AI 모델은 설문 응답에 적합하지 않다. 합성 응답이 인간 응답 패턴과 맞지 않기 때문이다"라고 인정하였다.

AI 페르소나 리서치 시장은 빠르게 형성되고 있지만, "사람처럼 반응하는가", "실제 제품을 사용하는가", "사람마다 다르게 반응하는가"라는 세 가지 핵심 질문에는 아직 답하지 못하고 있다. 이 공백이 곧 Personica의 진입 기회이다.


### ㅇ 창업 아이템의 국내·외 시장 현황 및 문제점

#### 시장 규모 — 깔때기형 축소

기업이 사용자를 이해하기 위해 쓰는 전체 예산을 합치면 글로벌 인사이트 산업은 $153B(약 214조원) 규모이다 (ESOMAR, 2025). 이 중 리서치 소프트웨어 영역만 $62B이며, 서비스($56B) 대비 2배 이상 빠른 11.5% 성장률을 보이고 있다. 사람이 수행하던 리서치가 소프트웨어로 전환되고 있으며, AI가 그 전환을 가속하고 있다.

Personica는 리서치 서비스 회사가 아니라 리서치 소프트웨어 회사이다. 아래 깔때기는 이 전제에서 출발하여, 각 단계에서 "왜 이만큼만 남는가"의 축소 근거를 함께 제시한다.

```
┌──────────────────────────────────────────────────────────────┐
│            글로벌 인사이트 산업  $153B (214조원)                 │
│            설문 + 인터뷰 + 분석 + 컨설팅 전체                     │
│                                          ESOMAR, 2025        │
└──────────────────────────┬───────────────────────────────────┘
                           │
          축소 ①  이 중 SW로 구매되는 설문 + UX 리서치만
                  (서비스·컨설팅·오프라인 조사 제외)
                           │
    ┌──────────────────────▼───────────────────────────┐
    │          TAM  $5.2B ~ $5.9B (7.2~8.2조원)          │
    │      온라인 설문 SW $4.7~5.3B  +  UX 리서치 SW $0.5B  │
    └──────────────────────┬───────────────────────────┘
                           │
          축소 ②  SW 시장 중 AI 페르소나가 침투 가능한 비율만
                  (설문 15~25%, UX 40~60%)
                           │
        ┌──────────────────▼───────────────────┐
        │       SAM  $0.9B ~ $1.7B (1.3~2.3조원)  │
```

### ㅇ 문제 해결을 위한 창업 아이템

위에서 확인한 세 가지 공백을 Personica는 동시에 해결한다.

| 시장의 공백 | Personica의 해결 | 결과 |
|------------|----------------|------|
| 긍정 편향 — "다 좋다"고 답함 | 감정 재현 엔진 — 좌절·불안이 수학적으로 누적되어 자연스러운 불만·포기 발생 | 더 사람다운 피드백 |
| 제품 미사용 — 상상으로 답함 | 실제 브라우저 사용 — AI 페르소나가 웹/앱을 클릭·입력·탐색한 후 피드백 생성 | 더 정확한 피드백 |
| 분화 부족 — "다 비슷하게" 답함 | 성격 기반 수학적 매핑 — Big Five 성격 모델에서 행동 파라미터를 도출하여 페르소나마다 다른 반응 생성 | 더 다양한 피드백 |

Personica는 "사람처럼 느끼고 판단하는 합성 페르소나가 리서치에 참여한 결과 리포트"를 산출한다. 서베이, UI/UX 리뷰, 제품 체험 3가지 모드를 제공하며, 서비스 구조와 개발 방법은 2장(실현 가능성)에서 상세히 서술한다.

## 2. 실현 가능성 (Solution)

### ㅇ 창업 아이템의 개발 계획

서비스 플로우 (4단계):

```
┌───────────────┐     ┌───────────────┐     ┌──────────────────┐     ┌───────────────┐
│  ① 고객 입력    │ ──▶ │ ② 페르소나 생성  │ ──▶ │ ③ 시뮬레이션 실행   │ ──▶ │  ④ 리포트 생성  │
│               │     │               │     │                  │     │               │
│ 질문/이미지/URL │     │ 타깃 기반 N명    │     │ 매 상호작용마다      │     │ 세그먼트별      │
│ + 타깃 유저 설명 │     │ Big Five 기반   │     │ 엔진 파이프라인 실행  │     │ 교차 분석       │
│               │     │ 세그먼트별 자동분화 │     │ 행동·감성 로그 수집  │     │ 감정 궤적 타임라인│
│               │     │               │     │                  │     │ + 개선 제안     │
└───────────────┘     └───────────────┘     └──────────────────┘     └───────────────┘

                              소요 시간: 합성 서베이 수 분 / 제품 체험(30명) ~40분
                              인간 리서치 대비: 2~4주 → 수십 분으로 단축
```

페르소나 리얼리티 엔진은 입력 방식에 따라 3가지 모드로 제공된다:

| 모드 | 입력 | 핵심 가치 |
|------|------|----------|
| 합성 서베이 | 텍스트 질문 | 설문·인터뷰 시뮬레이션 (제품, 브랜드, 시장조사 등) |
| UI/UX 리뷰 | 텍스트 + 목업 이미지 | 디자인 감정·인지·사용성 피드백 |
| 제품 체험 | 실제 URL + 계정 | 실제 제품 사용 행동·감정 변화 재현 |

#### 1) 아부 편향을 어떻게 해결하는가

Personica는 감정 계산을 AI(LLM) 외부에서 수학적으로 처리하는 방식으로 이 문제를 해결한다. AI가 감정을 스스로 판단하는 대신, 별도의 수학 파이프라인이 감정 상태를 계산하여 AI에 전달하는 구조이다.

| 비교 항목 | 일반 AI ("사용자 역할로 답해줘") | Personica |
|-----------|-------------------------------|-----------|
| 감정 변화 | AI가 직접 판단 → 긍정 편향 | 수학 파이프라인이 감정 계산 → AI에 전달 |
| 이탈 반응 | 거의 발생 안 함 | 좌절·불안 누적 시 구조적으로 이탈 발생 |
| 사용자별 차이 | 비슷한 답변 반복 | 성격에 따라 같은 오류에도 다른 반응 |
| 결과 | 과도하게 긍정적 피드백 | 실제 사람과 유사한 범위의 피드백 |

#### 2) 감정 변화 재현의 작동 원리 — 페르소나 리얼리티 엔진

각 AI 페르소나가 제품을 사용하는 매 순간마다, 다섯 개 레이어가 순서대로 실행된다. 각 레이어는 심리학·행동과학 분야의 검증된 이론 기반이다 (30편 이상 학술 논문).

```
    ┌─────────────────────────────────────────────────────────┐
    │  L1. 이 사람은 누구인가?  (Persona Profile)                │
    │  Big Five 성격 → 행동 파라미터 수학적 매핑                   │
    │  예: "꼼꼼한 성격(성실성↑) → 클릭 전 2초 망설임, 오류에 민감"  │
    └────────────────────────┬────────────────────────────────┘
                             ▼
    ┌─────────────────────────────────────────────────────────┐
    │  L2. 화면을 이해하는가?  (Cognitive)                       │
    │  Information Foraging, 인지 부하, 기대 vs 실제 괴리 계산     │
    │  예: "이 버튼이 어디 있는지 모르겠다 → 인지 부하 ↑"           │
    └────────────────────────┬────────────────────────────────┘
                             ▼
    ┌─────────────────────────────────────────────────────────┐
    │  L3. 지금 어떤 기분인가?  (Emotion)                       │
    │  OCC 감정 평가 → SDE 확률적 감정 진화 → PAD 연속 상태       │
    │  예: "오류 2회 연속 → 좌절감 0.3 → 0.6으로 상승"            │
    └────────────────────────┬────────────────────────────────┘
                             ▼
    ┌─────────────────────────────────────────────────────────┐
    │  L4. 다음에 무엇을 하는가?  (Decision)                     │
    │  BDI-E 모델 + Fogg 동기-능력 체크 + 만족화 게이트            │
    │  예: "좌절감 높음 + 목표 달성 불투명 → 뒤로가기 선택"         │
    └────────────────────────┬────────────────────────────────┘
                             ▼
    ┌─────────────────────────────────────────────────────────┐
    │  L5. 이전 경험을 기억하는가?  (Memory)                     │
    │  Memory Stream + Reflection + 습관 강도 + Drift Monitor  │
    │  예: "지난번에도 같은 문제 → 이번엔 바로 이탈"               │
    └─────────────────────────────────────────────────────────┘
```

#### 3) 실제 제품 사용 — 브라우저 자동화

제품 체험 모드에서 각 AI 페르소나는 실제 웹 브라우저를 조작한다. 매 스텝마다 아래 루프가 반복 실행된다:

```
    ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────────┐     ┌────────┐
    │ 화면 읽기  │──▶│이해도 계산 │──▶│ 감정 갱신  │──▶│   행동 선택    │──▶│  실행   │
    └──────────┘     └──────────┘     └──────────┘     │클릭/입력/스크롤│     └───┬────┘
         ▲            (L2 인지)       (L3 감정)        │ 뒤로가기/포기  │         │
         │                                             └──────────────┘         │
         └──────────────────────── 다음 화면 ───────────────────────────────────┘
```

#### 4) 개발 진행 상황 — "아이디어가 아니라 검증된 엔진이다"

| 모듈 | 상태 | 비고 |
|------|------|------|
| L1. 성격 기반 행동 파라미터 매핑 | 구현 완료 | Big Five → 클릭 망설임, 오류 허용도 등 |
| L3. 감정 재현 엔진 | 구현 완료 | OCC → SDE → PAD 수학 파이프라인 |
| L2, L4, L5 | 설계 + 일부 로직 확보 | 상용 수준 고도화 필요 |
| 브라우저 자동 조작 | 작동 확인 | 안정화 완료 |
| 제품 체험 시뮬레이션 통합 루프 | 초기 작동 | 30명 시뮬레이션 실행 완료 |

분화 검증 — Tally.so 대상 30명 시뮬레이션: 세그먼트별 평점 범위 3.50~4.33 (분명한 분화), 같은 제품에서 서로 다른 경로·이탈 이유 발생 확인.

현실성 검증 — G2 실제 리뷰와 비교:

```
  G2 실제 리뷰 일치율

  긍정 테마  (상위 10개 중 9개 일치)  ████████████████████████████████████████████████░░  90%
  불만 테마  (상위 7개 중 5개 일치)   ██████████████████████████████████████░░░░░░░░░░░░  71%
```

감정 엔진 효과 검증 — 이것이 Personica의 핵심 차별화 근거이다:

```
  감정 엔진 ON/OFF 비교

  엔진 OFF   이탈 반응 0건  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  ← "다 좋습니다" (비현실적)
  엔진 ON    자연 이탈 발생  ████████████████████████░░░░░░░░░░  ← 좌절·불안 누적 → 포기
```

#### 2) Personica의 포지션 — 경쟁사가 풀지 못한 공백

1장에서 분석한 세 가지 한계를 경쟁사별로 대조하면, 감정 재현과 제품 실사용을 동시에 해결하는 상용 서비스는 아직 없다. 아래 2×2 매트릭스에서 Personica는 유일하게 우상단에 위치한다.

```
          감정·인지·의사결정 시뮬레이션
                    ▲
               있음  │
                    │  Synthetic Users          ★ Personica
                    │  (Chain-of-Feeling,       (페르소나 리얼리티 엔진
                    │   감정 일부 도입)            + 브라우저 실사용)
                    │
         ───────────┼──────────────────────────▶ 실제 제품 사용
              없음   │                      있음
                    │
                    │  Simile · Aaru             Blok
                    │  (설문/인터뷰 대행,          (Figma + 이벤트 로그,
                    │   제품 미사용)                기존 데이터 의존)
                    │
               없음  │
```

Personica는 페르소나 리얼리티 엔진(감정 재현 + 성격 기반 분화)과 브라우저 자동화(제품 실사용)로 이 공백을 메운다. 학술적으로는 Amazon UXAgent(CHI 2025)와 Stanford Generative Agents(2023)가 방향을 검증하였으나, 이를 완전한 상용 서비스로 구현한 곳은 아직 없다.

#### 4) 단위 경제학 (Unit Economics)

원가의 대부분은 LLM API 비용이다. 페르소나 리얼리티 엔진의 레이어별로 경량 모델(60%)과 고성능 추론 모델(40%)을 혼합 배치하여 비용을 최적화한다.

| 레이어 | 역할 | 모델 | 근거 |
|--------|------|------|------|
| Perceive (DOM 파싱) | 페이지 → 관찰 | 경량 | 구조화된 파싱, 창의성 불필요 |
| Act (행동 실행) | 제품 조작 명령 | 경량 | 단순 명령 변환 |
| Memory/Reflection | 기억 저장·회상 | 경량 | 패턴 매칭 + 요약 |
| Emotion (OCC-PAD-SDE) | 감정 평가·진화 | 고성능 추론 | 감정 뉘앙스가 핵심 |
| Decision (BDI-E) | 행동 결정 | 고성능 추론 | 동기·갈등·포기 시뮬레이션 |

모드별 페르소나 1명당 원가(COGS):

| 모드 | LLM/명 | 인프라/명 | COGS/명 | 산출 근거 |
|------|:------:|:--------:|:-------:|----------|
| 합성 서베이 | $0.15 | $0.02 | $0.17 | ~15턴, 텍스트 응답만 |
| UI/UX 리뷰 | $0.26 | $0.04 | $0.30 | ~25턴, 이미지 토큰 추가 |
| 제품 체험 (30스텝) | $0.35 | $0.04 | $0.39 | Starter 기준 |
| 제품 체험 (100스텝) | $1.15 | $0.10 | $1.25 | Growth 기준 |
| 제품 체험 (200스텝) | $2.30 | $0.19 | $2.49 | Scale 기준 |

위 COGS/명은 시뮬레이션 비용만 포함하며, 리포트 생성 LLM 비용(건당 $0.30~$0.50)은 아래 매출총이익에 별도 반영되어 있다.

합성 서베이·UI/UX 리뷰는 전 티어 83%+ 매출총이익률 (SaaS 업계 평균 70~80% 상회). 제품 체험 Scale(50%)은 상대적으로 낮으나, LLM API 가격 하락 추세와 모델 라우팅 최적화로 마진이 자동 개선된다. 인간 리서치 플랫폼(User Interviews, Respondent.io) 대비 동일 30명 기준 1/7~1/10 비용이다.
