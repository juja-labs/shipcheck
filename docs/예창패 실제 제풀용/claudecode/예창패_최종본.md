## 1. 문제인식 (Problem)

---

### 1-1. 개발 동기

CB Insights 조사에 따르면 스타트업 실패 원인의 42%가 "시장이 원하지 않는 제품"이다. 미국 노동통계국 데이터에서도 민간 기업의 21.5%가 첫해에, 70%가 5년 이내에 실패한다. 문제는 제품을 만드는 것은 점점 쉬워지는 반면, 검증 방식은 10년 전과 크게 다르지 않다는 점이다.

| 스타트업 실패 원인 | 비율 |
|:-:|:-:|
| **시장 수요 없는 제품** | **42%** |
| 자금 소진 | 29% |
| 팀 문제 | 23% |
| 기술 부족 | 17% |

> 출처: CB Insights, 2024 (복수 응답)

[차트] 실패 원인 파이차트 — 42% "시장 수요 없음" 강조

| 구분 | 과거 | 2026년 현재 |
|------|------|-----------|
| 제품 제작 | 개발팀 필수, 수개월 | 노코드·AI로 1인 수일 내 출시 |
| 검증 방식 | 설문, FGI, 베타테스터 | **여전히** 설문, FGI, 베타테스터 |
| 검증 주체 | 전담 리서처 | PM 42%가 직접 수행 |
| 검증 비용 | UserTesting 연 $46K+ | 초기 팀에겐 여전히 부담 |

> 출처: Maze State of UX Research, 2025; Stripe Atlas, 2024

**대표자 문제 경험**: SSG.COM, SKT, KT에서 11년 8개월간 대화형 AI·생성형 AI 시스템을 개발하며 동일 문제를 반복 경험하였다. KT에서 20명 규모 팀을 리딩하며 보험·제조·교육·금융 등 다양한 도메인의 AI Agent를 개발했으나, 출시 전 사용자 감정·판단 예측은 어떤 경우에도 불가능하였다.

**핵심 문제**: "출시 전에 다양한 사용자 반응을 사람에 가깝게 시뮬레이션할 수 있는 방법이 없다."

→ Personica는 사람처럼 느끼고 판단하는 합성 페르소나(AI 가상 사용자)를 통해 이 문제를 해결한다.

### 1-2. 해결방안의 필요성

**1) 기존 사용자 검증 방식의 한계**

| 방식 | 장점 | 한계 | 비용·시간 |
|------|------|------|----------|
| 설문·인터뷰·FGI | 의견 수집 가능 | 실제 행동 확인 불가, 사후 합리화 편향 | 2~4주, 인당 $50~200 |

## 2. 실현가능성 (Solution)

---

### 2-1. 사업 개요 및 서비스 구조

**서비스 개요**: 사람처럼 느끼고 판단하는 합성 페르소나가 실제 제품을 체험하고, 감정 변화와 행동 로그를 포함한 리서치 리포트를 자동 생성하는 플랫폼.

**서비스 플로우 (5단계)**

| 단계 | 항목 | 내용 |
|:----:|------|------|
| ① | 고객 입력 | 스테이징 URL, 테스트 계정, 타깃 사용자 설명 |
| ② | 자동 분석 | 브라우저 자동화로 제품 크롤링 → 페이지/기능/플로우 파악 |
| ③ | 페르소나 생성 | 타깃 기반 N명 생성, 성격 모델 → 행동 파라미터 수학적 매핑 |
| ④ | 시뮬레이션 | 각 페르소나 독립 체험, 매 상호작용마다 페르소나 리얼리티 엔진 파이프라인 실행 |
| ⑤ | 리포트 | 세그먼트별 교차 분석, 감정 궤적, 이탈 지점, 개선 제안 |

[다이어그램] ①→②→③→④→⑤ 서비스 플로우 흐름도

**소요 시간**: 합성 서베이 수 분 / 제품 체험(30명 기준) 약 40분. 인간 기반 리서치 통상 2~4주 대비 크게 단축.

**3가지 시뮬레이션 모드**

| 모드 | 입력 | 제품 사용 여부 | 핵심 가치 |
|------|------|:---:|---------|
| 합성 서베이 | 텍스트 질문 | ✕ | 설문·인터뷰 시뮬레이션 |
| UI/UX 리뷰 | 텍스트 + 목업 이미지 | ✕ | 디자인 사용성 피드백 |
| 제품 체험 | 실제 URL + 테스트 계정 | ✔ | 실사용 행동·감정 변화 재현 |

**리포트 구성 항목**

| # | 항목 | 내용 |
|:-:|------|------|
| 1 | 세그먼트별 이탈 히트맵 | 유형별 이탈 위치 시각화 |
| 2 | 감정 변화 타임라인 | 쾌감·긴장·통제감 3축 궤적 |
| 3 | 세션 기록 | 화면별 사고 과정 및 감정 변화 기록 |
| 4 | 요약 평점 | NPS(순추천지수) 형태 |
| 5 | 개선 제안 | 시급도·영향도 기반 우선순위 정렬 |
| 6 | 페르소나 인터뷰 | 특정 페르소나에 추가 질문 가능 |

### 2-2. 핵심 기술 — 페르소나 리얼리티 엔진

Personica의 핵심은 **감정 계산을 AI(LLM) 외부에서 수학적으로 처리**하는 구조이다. AI가 감정을 직접 판단하면 긍정 편향이 개입하지만, 별도 수학 파이프라인이 감정 상태를 산출하여 AI에 전달하면 편향이 구조적으로 감소한다.

**일반 AI vs Personica 비교**

| 항목 | 일반 AI ("사용자 역할로 답해줘") | Personica |
|------|---------------------------|-----------|
| 감정 변화 | AI가 직접 판단 → 긍정 편향 | 수학 파이프라인이 계산 → AI에 전달 |
| 이탈 반응 | 거의 발생하지 않음 | 좌절·불안 누적 시 구조적으로 발생 |
| 사용자별 차이 | 비슷한 답변 반복 | 성격에 따라 같은 오류에도 다른 반응 |
| 결과 | 과도하게 긍정적 | 실제 사람과 유사한 범위 |

**페르소나 리얼리티 엔진 파이프라인 구조**

| Layer | 질문 | 기능 | 예시 | 학술 근거 |
|:-----:|------|------|------|---------|
| 1 | 이 사람은 누구인가 | 성격·숙련도 → 행동 성향 매핑 | 꼼꼼한 성격 → 클릭 전 2초 망설임 | Big Five, PRISM (Fudan 2025) |
| 2 | 화면을 이해하는가 | UI 복잡도, 기대-실제 괴리 계산 | 버튼 위치 불명 → 인지 부하 상승 | Information Foraging, TAM |
| 3 | 지금 어떤 기분인가 | 감정 변화를 수치로 추적 | 오류 2회 → 좌절감 0.3→0.6 | OCC Appraisal, PAD Model |
| 4 | 다음에 무엇을 하는가 | 감정+목표 기반 행동 결정 | 좌절감 높음 → 뒤로가기 선택 | BDI Model, Fogg Model |
| 5 | 이전 경험을 기억하는가 | 세션 간 학습·실망 누적 | 지난번 동일 문제 → 즉시 이탈 | Generative Agents (Stanford 2023) |

[다이어그램] 페르소나 리얼리티 엔진 파이프라인 세로 구조도 — 화면 입력 → L1~L5 순차 처리 → 행동 출력

**브라우저 자동화 실행 루프** (제품 체험 모드)

| 순서 | 동작 | Layer |
|:----:|------|:-----:|
| 1 | 화면 읽기 (현재 페이지 파악) | — |
| 2 | 이해도 계산 | L2 |
| 3 | 감정 갱신 | L3 |
| 4 | 행동 선택 (클릭/입력/스크롤/뒤로가기/포기) | L4 |
| 5 | 브라우저에서 실행 | — |
| 6 | 1번으로 돌아가 반복 | — |

→ 매 상호작용마다 이 루프가 실행되어 **최종 평점뿐 아니라 "왜 그런 결과가 나왔는지"를 시간 흐름에 따라 설명** 가능.

**학술 근거 종합**
